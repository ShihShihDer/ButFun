# 真實經濟設計 — 閉迴圈、防洗錢、漸進上線

> 願景來源:**玩家 2026-06-11**——「經濟越真實越好、防洗錢;NPC 身上要有乙太才能收購、
> 收的東西要能讓他賺;純賣商怎麼賺要想清楚;囤貨不玩的玩家怎麼辦」。
> 模型/機制:AI 規劃提案(目標是玩家的)。**選定:全套真實經濟、漸進上(玩家拍板)。**

## 核心鐵則:乙太不能無中生有,只能流動

把全服乙太想成水。健康經濟 = **水龍頭(注入) ≈ 排水孔(流出)**。
- **水龍頭**(乙太進玩家):打怪/採集掉落、任務、**NPC 收購**。⚠️ 現病灶:NPC 無限預算收購=水龍頭全開→通膨+倒貨套利。
- **排水孔**(乙太出):跟 NPC 買、買地、升級、合成、旅行、稅、**倉儲費**。
- **目標**:總注入 ≈ 總流出 → 物價穩、洗不出錢。

## 三種 NPC 怎麼活
1. **收購商**:**有限金庫**,壓低買(浮動價已有 ROADMAP 40)。靠轉手再賣/加工回血。金庫=硬約束。
2. **純賣商**(只賣不買):他是**排水孔**——收入=玩家乙太;貨來自上游供應商,他**付乙太進貨**(那筆流出經濟)。**純賣商不需要「賺」,他是把乙太抽離市場的基礎設施**,對抗通膨剛好有用。
3. **工匠**:又買又賣,賺「加工增值」差價。

## 防洗錢/防套利(讓無風險循環不可能)
- **買賣價差**(同 NPC 買 1 賣 2)→ 轉賣同一人不賺。
- **浮動價**(賣多買價跌、買多賣價漲)→ 洗市場自我抑制。
- **有限金庫**(收購商沒錢)+ **有限庫存**(賣商沒貨)→ 圈一下就卡。
- **時間回補**(價/金庫/庫存慢慢恢復)→ 不能快速重複。
- **AI 議價**:成交價**由引擎依市場算**,LLM 只在引擎允許區間點頭/殺價 + 有限金庫 + 離譜價上限 → 嘴再會講也洗不出錢。
- 合起來:每輪能榨出的乙太有上限、恢復很慢 → **沒有印鈔機**。

## 囤貨 / 不活躍玩家(玩家提問)
- **躺著不動的囤貨 = 穩定因子**(未流通,不造成通膨),不必動它。
- **唯一風險「回來一次倒光」**:被上面的「單筆內遞減 + 有限金庫」擋住——倒不出好價、掏不空更多金庫。**自我限制,炸不了經濟。**
- **為真實 + 促流動**(都顧資料安全):
  - **背包/倉儲上限**:身上不能無限囤,超量要蓋**倉庫**(花乙太=排水孔)。
  - **易腐品腐壞**:食物/作物放久不用會壞(自然排水孔、逼流動);**耐久品(礦/木/乙太)不壞=存款**。
  - **鐵律**:腐壞只套易腐品、是**透明遊戲規則**(非因不上線而偷扣);倉儲上限把超量**收進倉庫不是刪除**。**絕不懲罰性刪玩家資料。**

## 唯一旋鈕:小而受控的水龍頭(避免經濟鎖死)
金庫全封閉會鎖死(NPC 都沒錢→收不動)。留一個**緩慢「商隊收入」**幫金庫回血,調到「總注入 ≈ 總流出」。**這是唯一真正的新乙太來源,其餘全是轉移。**

## 漸進建造順序(一次一塊、每塊測試、走 review;**資料安全第一**)

> 資料安全鐵律:動到玩家乙太/背包,一律**向後相容 migration、不破壞既有資料**;金庫起始值給足,別讓玩家「突然賣不掉」;每塊先在不影響玩家的前提下加,再逐步收緊。

1. **商人有限金庫 + 收購從金庫付**(地基,防無限印鈔):每個收購 NPC 一個乙太金庫,ShopSell 從金庫扣;金庫不足→部分收/婉拒(「今天現金不夠,改天再來」)。起始值給足、緩慢回補(商隊收入)。**議價交易共用這個金庫。**
2. **AI 議價交易**(玩家實機發現的缺口):LLM 談妥夾 `[DEAL 品項 數量 價]`→引擎驗證(金庫/背包/引擎定價)→玩家確認成交。腦自由議價、手有界(只動有限金庫)。護欄:不無中生有、每筆/離譜價上限、不信 LLM 自報數字。
3. **修批量倒貨套利**:浮動價在**單筆內也遞減**(賣 600 不再全用市價,反映真實供需)。
4. **買賣價差**:NPC 買低賣高,殺掉同 NPC 套利。
5. **有限庫存 + 補貨**:賣商庫存有限、隨時間/供應鏈補貨。
6. **倉儲上限 + 倉庫**:背包上限、蓋倉庫(乙太排水孔)。
7. **易腐品腐壞**:食物/作物會壞(耐久品不壞)。
8. **供應鏈 + 純賣商進貨成本**:原料→成品分工;純賣商付上游進貨(乙太排水孔)。
9. **商隊收入平衡 + 經濟儀表**:受控注入旋鈕 + 監測水龍頭/排水孔平衡好調參。

防洗錢核心在 1~4 就成形;5~9 把真實感與閉迴圈補滿。
