# AI NPC 成長 Wave 2 — 讓 NPC 真正活在世界裡

> 願景來源:**玩家 2026-06-11 親口需求**(以下「玩家要的」皆指此)。
> 排序、技術選型、安全護欄:AI 規劃(標「我的提議」者為 AI 補充)。
> 上位藍圖見 [`VISION_AI_EMERGENT_ECOSYSTEM.md`](VISION_AI_EMERGENT_ECOSYSTEM.md);
> Wave 1(已完成 ROADMAP 58~71)見 [`PLAN_AI_NPC_GROWTH.md`](PLAN_AI_NPC_GROWTH.md)。

## 為什麼有 Wave 2

Wave 1 把六大 NPC 的「腦 + 記憶 + 手 + 派系湧現」全長完了(ROADMAP 58~71)。
但兩件事讓它還沒能真正上線、也還不夠「活」:

1. **效能牆(實測)**:本機純 CPU 上,一發塞滿脈絡的 NPC prompt 要 **~44 秒**,
   必然撞 20s 逾時退罐頭。多人同時聊只會更糟。**本機 CPU 服務不了豐富 NPC**。
2. **玩家要的「活世界」**:NPC 該彼此互動、有作息、有自己的經濟;新居民會遷入、
   要採買;城鎮會長大、有住家有糧食;獸潮來要有士兵迎戰;派系該有頭目;
   2D 人物該更精緻。**這些 Wave 1 都還沒有。**

## 七條安全護欄(沿用 Wave 1,不可破)

照 [`PLAN_AI_NPC_GROWTH.md`](PLAN_AI_NPC_GROWTH.md) 的七條:腦子自由手有界、
約束=世界規則(不能無中生有)、無寫死門檻(統計是「資料」給 LLM 判斷,不是 `if x>10 then 折扣`)、
玩家資料絕不毀(回歸乙太≠永久刪檔)、非同步不卡 15Hz 迴圈、降級+env gate、記憶防污染。

**Wave 2 追加兩條:**

8. **背景 NPC 生活預設不燒 LLM**:作息/移動/居民經濟以**模板與啟發式**驅動,
   LLM 只在「玩家在場且值得」時點綴。NPC↔NPC 閒聊**重度限流**。
   否則背景對話會把額度與 CPU 燒乾——這是規模化的頭號風險。
9. **美術只做原創**:精緻化走原創 sprite/程序生成,**絕不**抄任何外部遊戲美術進 repo
   (鐵律「AI 當老師不當影印機」)。

## 先決條件:LLM 雲端路由(必須最先做)

**為什麼最先**:沒有它,NPC 在 prod 根本回不了話(44s 逾時);而且 NPC↔NPC 背景對話
會把 LLM 負載乘上去——在效能修好前開背景對話=CPU 熔毀。所以**路由是其餘一切的地基**。

設計(在 `npc_chat.rs` 現有 `ollama_chat` 抽換點之後加 provider 路由):

- **降級鏈**:`雲端(快/並發) → 本機 ollama → 罐頭`。每個 tier 由「該 provider 的 key 是否存在」
  env-gate;沒 key 就跳過該 tier。**CI 無任何 key 也能綠**(走 ollama/罐頭路徑的純邏輯測試)。
- **Provider(OpenAI 相容 client,reqwest)。皆為「真免費層、獨立 key,與維護者的
  Gemini 訂閱無關」**(澄清:迴圈用的 `gemini` CLI 走的是**維護者付費訂閱**,不是免費,
  NPC 路由**不可**借用它):
  - **Groq**(主力):免費層、LPU 超快、server 端天然並發、零本機 CPU。`GROQ_API_KEY`。
  - **OpenRouter**(個性多樣化):一把 key 接多個 `:free` 模型 → **不同 NPC 配不同模型=
    免費個性差異**(玩家先前想要的)。`OPENROUTER_API_KEY`。
  - **Cloudflare Workers AI**(備援):免費層、server 端執行。`CF_*`。
  - **Gemini AI Studio**(選用):另有一個**免費 API key**(generativelanguage.*),
    與付費的 gemini CLI 是兩回事;額度小,可有可無。`GEMINI_API_KEY`。
- **prompt 瘦身**:世界新聞/人際/派系等脈絡層只注入「最相關的幾條」+ 硬字數上限;
  小 prompt = 延遲/成本到處都降(本機與雲端皆然)。
- **每 NPC 綁模型**:設定哪個 NPC 走哪個 provider/model,個性自然分化。
- **成本/安全**:每 provider 限流 + 回覆快取/去重;`tokio::spawn` 不卡迴圈;
  key 一律 `.env`(gitignored,**由人放入**,絕不進 repo)。
- **驗收**:有雲端 key 時,多人並發聊各自 <2s 回 AI;無任何 key 時優雅降級(CI 綠)。

> **只有人能做的一步**:申請一把免費 key(推薦 Groq,約 2 分鐘)放進 `.env`,
> 再把 `BUTFUN_NPC_LLM=1` 開閘。路由本身迴圈可全部蓋好;**啟用雲端=人放 key + 開閘**。
> 在那之前,prod 的 NPC 維持秒回罐頭(安全、零 CPU 風險)。

## 建造順序(ROADMAP 72~80,一次一塊、別跳級)

照 ROADMAP.md 由上往下,每塊獨立可上、附測試、玩家看得到:

1. **72 LLM 雲端路由 + prompt 瘦身**(先決條件,見上)。
2. **73 NPC 作息與在場**:NPC 在世界裡走動、有日常行程(模板/啟發式驅動,**不per-step呼叫LLM**)。玩家看得到鎮上有人在動。
3. **74 NPC 之間自發閒聊**:彼此偶爾對話,玩家旁觀「世界是活的」(重度限流、走雲端路由、多為點綴)。**依賴 72**。
4. **75 新居民遷入**:人口隨繁榮自然增長(湧現,綁資源,非寫死人數),新居民會去採買。
5. **76 NPC 的家與生計**:住家、存糧、吃飯;城鎮規模隨之長大(玩家看得到新建築)。
6. **77 NPC 自有經濟**:居民之間互相買賣,價格隨供需浮動(延續「約束=世界規則」,有限存量)。
7. **78 獸潮防衛**:士兵 NPC 迎戰怪物浪潮、守護村莊(獸潮事件已存在,見 ROADMAP 69)。
8. **79 派系頭目自然浮現**:影響力高的 NPC 成為派系核心——**從好惡/影響力湧現,不寫死頭銜**(延續 ROADMAP 71)。
9. **80 2D 角色美術精緻化**:原創 sprite 讓人物更好看(護欄 9:只做原創)。

每塊完成在 ROADMAP.md 打勾並簡述;走審查閘(`dev2review.md`),**非緊急不自合**。

## 湧現,不是規定(再強調一次)

居民經濟、派系頭目、人口、結盟對抗——都給「需求 + 世界規則」,讓它**自然長出來**,
不要寫死「這個是國王」「滿 10 次給折扣」。玩家要的就是「分久必合、合久必分」自己發生。
設計師介入愈少愈好。
