# 視覺精緻化計畫 — 讓世界漂亮一點(但要順)

> 願景來源:**玩家 2026-06-11 親口需求**——「想要美化遊戲,讓人物、怪、背景漂亮一點,
> 可能加一點點渲染;**前提還是要讓它順順地動**;手機現在蠻厲害的,想讓整個世界更美、
> 甚至有編輯系統、更廣闊」。結構/鐵律/取捨:AI 規劃(標「我的提議」者為 AI 補充)。

## 四條鐵律(不可破)

1. **效能優先**:手機中階機要維持流暢。特效**分品質級**(高/中/低),依裝置能力自動選或讓玩家切;
   弱機自動降級(關視差/關柔影/退回簡圖)。**絕不為了漂亮犧牲流暢**——這是玩家的明確前提。
2. **原創美術**:所有 sprite/背景自己畫或程序生成,**絕不抄任何外部遊戲美術**(鐵律「AI 當老師不當影印機」)。
3. **守 Canvas 2D 為預設**:環境光、柔影、粒子、視差、漸層光暈這些 **2D 特效足夠**先讓世界變美。
   **換 WebGL / 引入渲染引擎是架構級決策——迴圈不可自行跳,要先 `for_human.md` 升級給人拍板。**
4. **純前端視覺**:只動畫面呈現,**不動玩家資料 / 不改遊戲邏輯 / 零 migration**。

## 安全可做的順序(ROADMAP 88~91,一次一塊)

1. **88 角色 sprite 精緻化**:原創蒸汽龐克小人(分件、沿用換色),弱機退簡圖。前後對比附 PR。
2. **89 怪物 sprite 精緻化**:各生態與精英/怪物王獨特原創造型,威脅一眼可辨。
3. **90 環境光與日夜光暈**:Canvas 疊光源柔光、日夜色溫、柔影,效能預算內、可開關。
4. **91 生態背景層次與視差**:遠景剪影/雲霧/光帶 + 輕量視差,先用視覺把「世界更廣闊」做出體感。

每塊:玩家一眼看得到變漂亮、附前後對比截圖、效能不退、原創美術、`cargo test` 綠、走 review 不自合。

## 要先跟人定義的大方向(迴圈別自行開架構)

這些**玩家有興趣但還在發酵**,迴圈遇到時**寫進 `for_human.md` 升級給人**,別自己決定:

- **世界編輯 / 建造系統**:玩家擺放裝飾、蓋自己的家——這是**新玩法系統**(不只視覺),
  範圍大、要定義玩法與持久化,先跟人談清楚。
- **WebGL / 重渲染**:架構級,效益 vs 複雜度 vs 手機相容要評估,先問人。
- **「世界更廣闊」的具體形態**:世界已是無限地形(見 `PLAN_INFINITE_WORLD.md`);
  「更廣闊」可能是更多生態/地標/尺度感/可去的地方——具體做什麼先跟人收斂。

## 為什麼先視覺、後大系統

先把**安全、玩家立刻有感、不傷效能**的視覺精緻化做起來(人物/怪/背景/光),
讓世界肉眼可見地變美;編輯系統與重渲染這種**大而需取捨**的,留給人一起定義方向再動,
符合「寧可小而完整,不要大而半成品」的專案紀律。
