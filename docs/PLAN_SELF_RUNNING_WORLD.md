# 階段轉向：優化深度 + 自走世界（玩家 2026-06-11 戰略）

> 願景來源：**玩家 2026-06-11**——「現階段排程的東西做完後，該回頭好好優化：
> ①既有功能的合理性 ②畫面/美術 ③NPC 自主性——讓這變成一個**真的會運轉的小世界**，
> **就算沒有玩家，也能靠經濟系統 + NPC 的 AI 自己運轉。**」

## 階段轉向（不是停，是換檔）

目前是「衝功能」階段（迴圈全速長 ROADMAP）。**等手上排程的東西做完、衝到一個里程碑後，
換檔到「深度」階段**：少疊全新功能，多做下面三件——

### 1. 功能合理性（把既有系統接起來、補破綻）
- 像「下雨卻不澆田（109）」那種**系統各做各的、玩家一眼看穿不合理**的洞，系統性盤點、接起來。
- 迴圈可定期做「合理性巡檢」：找出「A 系統理應影響 B 系統卻沒有」的斷點，補上。

### 2. 畫面/美術（與功能優化同時進行）
- 角色/怪/建築/環境光/天氣 持續精緻化（見 `PLAN_VISUAL_POLISH.md`、`PLAN_MULTI_RENDERER.md`）。
- 與①並行：優化功能合理性的同時，順手把相關畫面也做好看。

### 3. NPC 自主性 → 自走世界（最重要、最有辨識度）
**目標：0 玩家在線時，世界仍持續運轉**——NPC 過日子、彼此交易、經濟循環、需求/關係/派系自然演化。

- **現況阻礙**：`src/game.rs` 有 **17 處** NPC 行為被 `online_count > 0` 卡住（廣場夜談、工位對話、
  各種 NPC 事件/委託/懸賞/加成…）。**沒玩家 → 世界凍住**。
- **方向**：選擇性鬆開這些門檻，讓世界在無人時也演化——但**嚴守成本紀律**：
  - **背景模擬一律用便宜的啟發式/模板/純邏輯**（NPC 作息移動、經濟轉移、需求/關係/派系漂移與偵測）——
    **0 玩家時近乎不呼叫 LLM**（LLM 是錢/速率）。
  - **NPC↔NPC 經濟**：NPC 之間自己買賣、把貨與乙太在供應鏈上流動，**即使 0 玩家也循環**（接 `PLAN_ECONOMY_REALISTIC.md`）。
  - **LLM 只在**：玩家在場時的對話/點綴、或玩家回來時的「你不在時發生了 X」追述、或極稀疏的背景風味（重限流）。
- **驗收**：掛 0 玩家放著一段時間，回來看世界**真的變了**（價格波動、NPC 關係/派系變化、有人致富有人落魄、
  庫存與金庫流動）——而這段時間的 LLM 花費**趨近於零**。

## 鐵律
湧現不寫死、背景不燒 LLM、原創美術、資料安全、一次一塊、走 review。
這階段**寧可把已有的做深做合理，也不要再鋪一堆半成品新功能**——符合專案「小而完整」紀律。
