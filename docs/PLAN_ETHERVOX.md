# 乙太方界 — 由 AI 棲居的 voxel 世界（路線圖 + 願景）

> **北極星**：一個 voxel（麥塊風）世界，AI 居民**真的住在裡面活著**——記得、形成關係、有持久的渴望、會蓋東西、長出一個**會自己演化的小社會**；人類是訪客／園丁，看著它長。
> 維護者 2026-06-30「全權給你執行、v2 與未來慢慢做下去」。Rust 後端權威、前端只渲染；保留並逐步搬入 AI 大腦/禱告/帳號/經濟。舊 2D/3D 靈魂搬齊後封存。

## 核心信念（執行時的指北針）
- **記憶不是功能，是讓居民「真的活著」的土壤。記憶要驅動行為，不只用來聊天。**
- **先把「有感的魔法」做出來**（記得你 + 你的互動真的有後果），重 infra（向量/壓縮）等規模需要時再上。別為完美 infra 延後那個「哇」的瞬間。
- 每刀都要能讓人「看得出世界更活了」。FPS 鐵律：大量實體一律 InstancedMesh（見 #614/#820）。動 FPS/手感一律用真瀏覽器 QA（`scripts/qa/browser-qa.mjs`）、voxel 用直式測。

## 路線圖（一刀刀慢慢長，每刀真機驗證+我審）
1. **記憶 v1（進行中）**：每居民**獨立持久記憶庫** + 對話歷史 + 回想（記得你/聊過什麼/重開還在）。
2. **記憶 v2**：**向量語意檢索**（deb-pc ollama `bge-m3` 免費本機嵌入）+ **整併/壓縮/封存**（reflection：把舊記憶摘要成高階印象；很舊的封存）。importance 加權檢索（相關+近期+重要）。
3. **記憶→行為**：居民有**持久目標/渴望**（露娜記得想要家 → 幾天內採方塊蓋小屋）；**你的互動有後果**（幫過牠→對你更好/送東西）。禱告從一次性句子變成持久渴望。
4. **居民↔居民關係**：居民記得彼此、有關係（熟識/幫過/吵過）→ 友誼、合作、小圈子**自己湧現**＝小社會。
5. **日記/生命故事**：每居民的記憶累積成一本**人類可翻閱的日記**——看牠這幾天想什麼、遇到誰、蓋了什麼。把「AI 的內在生活」變成訪客最有感的東西。
6. **禱告驅動蓋家/設施**：居民用 delta 系統把渴望蓋成真（承接諾娃古井精神，但由居民自己持久目標驅動）。
7. **居民散佈世界各處住**（別擠主城，麥塊式散居）。
8. **搬帳號/經濟進乙太方界 → 舊 2D/3D 封存**（先搬靈魂再封、不刪、可復原）。

## 玩家遊玩（讓人類也享受——與 AI 生活並行的另一條主軸）
維護者 2026-06-30：「記得操作/合成等系統也要，畢竟我也想享受這世界。」**乙太方界不只是給 AI 住的展示櫥窗，也要是「人類自己玩得爽」的世界。** 兩條並行、互相加分：
- **操作手感**：MCPE 風觸控持續打磨（已有準心/破壞/放置/搖桿/直式；feel 要順）。
- **合成（crafting）**：採集材料 → 合成方塊/道具/工具（把 2D 的合成概念帶進 voxel）。
- **採集/資源**：砍樹、採礦、撿拾——voxel 資源節點。
- **種田**：在 voxel 裡種/長/收。**蓋造**：更多方塊型別、家具、裝飾。
- **經濟/交易**：跟居民交易、共享經濟。**進度/目標**：療癒循環（採集→合成→蓋造→與居民同住）給人想一直玩下去的理由。
- **交織點**：你採集+合成+蓋造，居民在同個世界活著+蓋家，你們交易/互助/送禮——**人類的樂趣與 AI 的生活在同一片方塊天地交織**，這正是它比純 AI sim 或純建造遊戲都更迷人的地方。

## 鐵律
向後相容、絕不破壞玩家資料；守鎖紀律（LLM/記憶讀寫別在持鎖中 await）；複用同一份 world-core；機敏走 env；不抄外部碼。腦池：DNS 已修，Groq 多 key 為主、Cerebras/Gemini 後備；便宜大腦命門長期仍要顧。

## 對話 / 社交 / 內心系統設計（2026-06-30 與維護者共同定，建構時必守）
- **embodied 空間化**——不是單窗、不是頻道：對話用世界裡的**對話泡泡**呈現、不蓋住世界；底部留最小輸入列+快捷句。
- **範圍說話**：說話有半徑，範圍內居民「聽到」。**對誰講＝走近/面對誰**，不用 @。雙向：居民彼此也範圍說話，玩家走過會**旁聽到**（世界有人聲、活著）。
- **內 vs 外兩層**：居民有「**內心流**」（思考/心情/渴望/許願＝**私密**，驅動行為+記憶+日記）；說話只是浮出的一小片。**日記＝瞥見沒說出口的內心。**
- **聽到 ≠ 要回**：聽到**默默進記憶（零 LLM）**，多數時候只聽不講。
- **「要不要回」＝便宜閘（零 LLM）**：是否被直接對話 + 個性(外向度) + **戳中度**(碰到興趣/記憶/渴望→關鍵) + 心情/狀態 + 冷卻/場景預算 → 機率 → 擲骰。**「回什麼」才動 LLM。** 戳中度吃記憶+渴望 →「她一聽到星星就眼睛一亮」，活而不機械。
- **防對話風暴三閘**：①每居民說話冷卻 ②場景級說話預算 ③機率×個性。居民彼此短對話+冷卻，不 reply-storm。
- **念頭播種閉環**：你的話被 overhear → 進居民記憶 → 若戳中個性 → 起念 → 許願 → 造世界 AI 實現（**優先變成「讓人和居民都能用的配方/系統」**，如沙+熱→玻璃，而非憑空生物件）。人類當園丁種念頭。
- **非同步層**：城鎮日記牆/動態 feed——不在時回來能讀「居民說了什麼、做了什麼、許了什麼願」。
- **成本鐵律**：聽是免費（只進記憶）、只有真的開口才花 LLM。

## 合成系統設計（2026-06-30 維護者，建構時必守）
維護者：「合成做成更像 Minecraft；背包簡單合成、複雜的要工作台；不然通通做成按鈕太複雜；有現成好介面可抄、多多善用。」
- **參考 Minecraft 合成 UX（現成、玩家熟、抄設計不抄碼）**——**別「一個配方一個按鈕」**（recipe 一多就亂、不可擴充）。
- **分層**：
  - **背包簡單合成 = 2×2 格**：背包介面內建 2×2 合成格 + 結果格，做簡單配方（如木頭→木板）。
  - **複雜合成 = 工作台 3×3 格**：放置「工作台」方塊、互動開 3×3 格 + 結果格，做複雜配方。
- **格子式擺放 + 配方比對**（有形 shaped / 無形 shapeless，位置可有意義）——這樣 recipe 無限多也不亂、好擴充。「沙+熱→玻璃」這類「世界學會的配方」也走這套格子系統。
- **手機/直式友善**：點背包物品→點格子放（或拖曳）；**現有合成的直式重疊 bug 要修**。
- 現有 #853「合成台 v1（按鈕式木板/石磚/玻璃）」**演化成這套格子式**，別再加更多按鈕配方。
