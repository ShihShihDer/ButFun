# 可挖地形 + 建造（Core Keeper 風・俯視）設計藍圖

> 使用者 2026-06-09 拍板的**核心風格**：**Core Keeper 那種俯視（top-down）挖掘**——能**挖通**地形、
> 把挖到的材料**堆回去建造**，做出「洞穴家 / 洞穴城堡」。整個遊戲從療癒農場往「挖掘 + 建造」長。
> 這是北極星玩法（ROADMAP「接著 6」）。**大切片、架構級**，照本文件由上往下、一次一片。
>
> **視角（拍板，別搞錯）**：**俯視 top-down**——玩家在地圖上走動、挖周圍的牆與地下；格子就疊在
> 玩家走的這個平面上。**不是缺氧那種側視剖面 / 重力往下挖**。2026-06-09 使用者明確選 Core Keeper。

## 與現有引擎的關係（重要）

現在的世界是「連續平面上擺實體（玩家 / 怪 / 礦點 / 田）」。本功能新增一層**格子地形（tile world）**
蓋在同一個座標系上，不是打掉重練：

- 每個 chunk 一張 tile grid（**沿用現有 chunk 分塊與確定性生成**那套）。
- 每格一個材料 enum（`Empty / Dirt / Stone / Ore / …`）。
- 初始地形**確定性生成**（同 seed → 同地形，不必存整個世界）；玩家的挖 / 建只存**差異（delta-save）**：
  稀疏對應 `(chunk, cell) → 覆蓋材料`，落 Postgres。**這正是 ROADMAP 原本講的「delta-save 地形」。**

## 互動模型

- **挖（Dig）**：WS 訊息 `Dig{cell}`。可及範圍內、目標是實心 → delta 設 `Empty` + 對應材料掉進背包 +
  廣播。沿用現有 Gather / Farm handler 的鎖紀律（**不可在持 `players.write()` 時去拿 tile 鎖**）。
- **建 / 放（Place）**：WS 訊息 `Place{cell, material}`。目標 `Empty`、可及、背包有該材料 → delta 設實心 +
  廣播。這就是「把挖到的土堆回去蓋牆」。
- **碰撞**：實心格擋移動、`Empty` 可走——這是讓「洞穴 / 牆」真的有意義的關鍵，也是最動到既有移動邏輯的一刀。

## 與農耕的關係

先**並存**：地表開放地照舊耕種（田 / 地塊不動），挖掘針對「牆 / 地下 / 礦脈」。
之後（選配）再考慮把「土壤」也納入 tile 型別，讓農耕與地形統一——那是較大重寫，**不在首版**。

## 切片（additive-first，每片可獨立上線、reviewer 逐片把關）

- **C-1（純加法、不破壞）**：tile 層資料結構 + 確定性生成 + delta-save 持久化（新增表，向後相容）+
  AOI 廣播 + 前端畫出色塊地形。**先不碰碰撞、不可挖**——只是「世界多了看得到的地形方塊」。可獨立 merge。
- **C-2 挖**：`Dig` handler（實心 → `Empty` + 掉材料）、可及性檢查、廣播、前端點擊挖。此時尚無碰撞，
  挖 = 採集 + 開路的視覺與掉落。
- **C-3 碰撞（最危險的一刀）**：實心格擋移動。動到移動主迴圈 → **合併前一定要過冒煙閘**
  （起 server → 連 ws → 收到含自己的快照 → 遊戲迴圈活著），並測「卡進牆內 / 被埋住 / 出生在實心格」邊界。
- **C-4 建造**：`Place` handler（背包材料 → 實心格）。至此可堆土蓋洞穴家 / 洞穴城堡。
- **C-5（選配、之後）**：結構件（門、地板、光源）、農耕併入 tile、洞穴專屬生態 / 礦脈深度。

## 護欄

- 每片附**純邏輯測試**（生成決定性、delta 套用順序、挖 / 建規則、可及性判定）抽成可測函式。
- **不抄外部遊戲碼**（只吸收概念寫原創碼）。面向玩家字串留 i18n 空間、註解繁中。
- 動玩家資料走 migration、**向後相容**（新增 delta 表，不動既有表）。
- 每片都要讓玩家「看得出世界長大了」；C-1 看得到地形、C-2 挖得動、C-3 走得進洞、C-4 蓋得出家。

# Phase 2：正宗 Core Keeper（實心世界・挖隧道）

> 使用者 2026-06-09 playtest 後拍板：地形要**反過來**——不是「開放地上撒零星土塊」（兩邊缺點：
> 走路一直撞散塊、又不像洞穴），而是 **Core Keeper 那種「整片實心、你主動挖隧道開路」**。
> 開放區 = 新手村 + 天然洞窟 + 你挖出來的路。怪與資源只待在開放區，不會被埋在石頭裡。

## 反轉後的世界模型

- `tile_kind_at`：**安全區外，預設實心**（不再是預設 Empty）。實心材質依生態域 + 深度：
  Dirt（好挖、常見）/ Stone（硬、岩域多）/ Ore（稀疏礦脈，小簇）。
- **天然洞窟**：用一層**低頻雜訊**（粗網格的平滑 hash）在實心中挖出**連通的開放洞窟/走廊**
  （Empty），讓玩家有天然空間探索、移動，不是 100% 封死。密度可調：起手約 **65–75% 實心**
  （要能挖出路、又不會走兩步撞牆），實際靠 playtest 微調。
- 安全區（新手村）永遠 Empty（已在 PR #64 做）。前端 `tileKindAt` 同步同一套生成 + 洞窟雜訊。
- 礦脈（Ore tile）就嵌在實心裡——你**挖到才拿得到**，這就是 Core Keeper 的採礦。

## 切片（**順序很重要**：先讓實體會避開實心，再反轉，否則怪會被埋住卡死）

- **D-1（先做・安全）：怪 / 資源「只生在開放格」**。敵人與採集節點的生成座標若落在實心格，
  就往外找最近的 Empty（洞窟）格放；找不到就略過該次生成。**先上這片**（現在地形還稀疏、無害），
  之後反轉才不會把怪埋進石頭、被 #64 的碰撞卡死。附測試：實心格的 spawn 會被搬到 Empty。
- **D-2：反轉地形為「實心 + 洞窟」**。`tile_kind_at`（與前端）改成安全區外預設實心 + 洞窟雜訊。
  調密度到「好挖、好探索」。附測試：安全區仍 Empty、洞窟雜訊決定性、某些座標為實心。
- **D-3：導航 / 小地圖**。修小地圖「像素隨移動變形」（次像素對齊：地形取樣對齊世界整數格、
  不隨相機抖）；小地圖畫出實心/洞窟/已挖路 + **新手村「家」標記**，讓人在隧道迷宮裡找得到路回家。

## 與既有系統的銜接

- 農耕：地表開放的安全區照種；underground/wilderness 是實心可挖層。首版**先並存**，不強求統一。
- 既有 gather 節點（樹/石）：D-1 後會被搬到開放洞窟；長期可考慮把「礦」完全交給 Ore tile、
  樹留地表——但**不在首版**，別擴張。
- 效能：地形仍是**確定性本地生成 + 只廣播 delta**（C-1 設計），實心變多只是多畫色塊，不爆頻寬。
- 玩法迴圈完成後長這樣：出生在開放新手村 → 挖進實心大地 → 找到洞窟/礦脈/怪 → 挖出空間 →
  用挖到的材料蓋牆（C-4）→ 洞穴家。**這就是使用者要的 Core Keeper。**
