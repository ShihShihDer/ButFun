# ButFun 遊戲設計文件（GDD）

> 狀態：概念方向書 v3（討論稿）。本文件記錄遊戲願景與設計方向，**尚未開始實作程式碼**。
> 目的：把零散的點子收斂成一份可被執行、可被砍範圍、可被一個人＋AI 扛起的設計。

---

## 1. 願景一句話

> 一個在瀏覽器就能玩、手機電腦皆宜的**蒸汽龐克太空歌劇**療癒多人世界：
> 從一顆星球的一塊農莊開始，靠「乙太燃料經濟」把種田、自動化、採集建造、職業社交、
> 開車競速串成一個會互相餵養的循環，並朝「海陸空外太空、多星球星際」的方向生長。

靈感來源：星露谷物語、牧場物語、初代 RO、缺氧（Oxygen Not Included）、麥塊（Minecraft）、
極速快感 / 極限競速 地平線、初代 GTA（2D 俯視）、星銀島（Treasure Planet）。

---

## 2. 世界觀：蒸汽龐克太空歌劇（Steampunk Space Opera）

《星銀島》式的設定：以**乙太（Aether）/ 蒸汽動力**驅動的船，能從海面航行 → 升空 →
穿越星際虛空 → 在星球之間旅行。科技是齒輪、黃銅、壓力管路、蒸汽與一種叫「乙太」的
魔法燃料的混合體。

- **多星球＝主題化地圖**：沙漠機械星、叢林乙太星、冰封礦星……天然支援 RO 式多地圖擴充、
  星露谷式的生態與季節變化。
- **駕駛分三層**：星球地表 2D 俯視開車 / 卡丁車；星際間飛船航行探索；競速 = 地表卡丁賽
  +（未來）星軌飛船賽。
- **經營分散各處**：在母星基地、飛船、殖民地種田 / 自動化 / 採集 / 建造，隨星際擴張開新殖民地。
- **燃料經濟貫穿全部**：乙太燃料驅動機台（缺氧式自動化）、地表載具、星際飛船。

### 2.1 背景故事：「大靜默」之後的拓荒

星系裡的一切都靠乙太運轉。曾經有過一個乙太文明的黃金時代，直到「**大靜默**」——
乙太的流動突然崩潰：天空之船墜落、自動機械沉睡或失控、殖民地之間的航線被切斷、
許多世界被遺忘。

如今，乙太正緩慢回流。拓荒者重返散落的星球，在舊時代的廢墟上重新建立生活。

這個前提一次合理化所有系統：

| 系統 | 被故事合理化為 |
|---|---|
| 採集殘骸 | 撿拾墜落的舊時代科技 |
| 打自動機械 | 對付沉睡 / 暴走的舊機械 |
| 種乙太作物 | 乙太珍貴，需靠農業培養回流 |
| 多星球隔離 | 殖民地航線被切斷，待重新連接 |
| 療癒拓荒調性 | 重建家園，而非征服 |

## 3. 開場體驗（也是 MVP 的教學）

> 注意：開場依**種族起源**而不同（見第 5 章）。以下是 **MVP 採用的「地球人 / 繼承農莊」起源**。

開場約 10 分鐘：

1. 抵達邊境星球的港口（介紹世界）。
2. 走向你繼承的地盤：前人留下的**生鏽乙太工坊** + **荒蕪乙太田** + 角落一艘**壞掉的飛船**
   （星露谷「繼承農莊」鉤子的蒸汽版）。
3. 沿路看到其他**真實玩家**在移動（多人臨場感）。
4. 整地、種下第一株乙太作物、澆水。
5. 告示板 / NPC 指引你修復精煉機。
6. 日夜推進，作物長成、收成，得到第一份乙太。
7. 飛船標示「**修復進度 0%**」——埋下星際旅行的北極星勾子。

## 4. 乙太燃料經濟（設計靈魂）

這是讓「六款風格」咬合的核心，不是把六款並列，而是讓它們**互相餵養**。

### 4.1 四層資源

- **乙太（Aether）**：魔法燃料 / 世界的血液。
- **礦石（Ore）**：工業原料。
- **作物（Crops）**：食物 / 乙太作物。
- **零件（Parts）**：礦石 + 乙太鍛造而成。

### 4.2 核心循環

```
農場種乙太作物 / 食物
      │
      ▼
工坊機台精煉（吃煤 / 水 / 壓力，缺氧式）
      │
      ▼
精煉出 乙太 + 零件
      │
      ▼
驅動：自動化機台 / 地表載具卡丁車 / 星際飛船（未來）
      │
      ▼
競速 / 探索 / 採集（消耗燃料、產出零件與獎金）
      │
      ▼
回頭餵農場與工坊（循環閉合）
```

### 4.3 設計要點

- **乙太＝「快樂的貨幣」**：每個活動非產出即消耗乙太 / 零件，玩任一支柱都推進其他支柱。
- **天然製造玩家互賴**：車手要農夫的燃料、農夫要車手的稀有零件、工程師用自動化省時去社交
  ——這是 MMO 社交黏著劑，單人時則是進度主軸。
- **健康經濟**：產出（種田 / 採礦 / 競速獎勵 / 交易）vs 消耗（機台燃料 / 載具燃料 / 鍛造升級 /
  建造）。沒有無限資源，必須持續運轉農場 / 精煉（療癒日常）來供應冒險（開車 / 競速 / 探索）。
- **一天的循環**：早上澆水收乙太 → 中午開車採集 / 競速 → 下午鍛造升級擴自動化 →
  傍晚進城社交交易。
- **MVP 只取最小一截**：一種乙太作物 → 一台精煉機 → 產出乙太，先證明「種 → 精煉 → 用」好玩。

## 5. 種族「起源（Origin）」系統

仿《闇龍紀元：起源》（Dragon Age: Origins）/ 柏德之門的 Origin 設計：
**世界是共享的，但每個種族 / 出身有自己專屬的開場序章**，反映其文化，然後匯流進同一個
持續世界。

種族走**原創星際人種**路線（不用老套的精靈 / 矮人 / 機器人），各有母星、文化、特徵。

### 5.1 資料驅動架構

每個種族 = 一筆資料：

```
{
  母星,
  文化,
  特徵 / 主題小特性 (trait),
  專屬起源序章 (origin scenario),
  起手玩法傾向,
  立繪 / sprite
}
```

引擎只讀一份「種族清單」。**新增種族 = 新增資料檔 + 開場腳本 + 美術 + lore，不動核心程式**；
lore 可日後逐步補。這與「多星球北極星」是同一招：先蓋可擴充插槽，內容當更新逐個塞。

### 5.2 種子設定（起點，可擴充）

| 種族 | 母星 / 文化 | 專屬起源開場 | 玩法傾向 |
|---|---|---|---|
| **地球人（索爾人）** | 綠意農業星，務農工匠之民 | 繼承荒廢工坊農莊（療癒鉤子） | 種田 / 經營 |
| **施拉雅人** | 無固定母星，住在星際商會飛船船隊上的遊牧商賈 | 商船學徒，船隊停靠邊境星，下船自立門戶 | 貿易 / 移動 |
| **布司人** | 重工業礦星，齒輪與熔爐之民，矮壯耐勞 | 老家礦坑塌陷，帶著工具出走，來邊境重開礦脈 | 採礦 / 自動化 |
| **燐光族** | 永夜的乙太霧星，與乙太共生、會發光的煉金祭司 | 循著乙太回流的徵兆，受召而來 | 乙太 / 精煉，夜視 |

> 以上名稱與設定只是種子，之後增刪都很便宜，不必現在定死。

### 5.3 範圍紀律

- **MVP 只做「地球人 / 繼承農莊」一個起源**，但**創角 / 起源系統從一開始就做成可插拔**；
  其他種族 / 起源當內容更新逐個加。
- 種族特性**刻意做小**（外觀 + 主題小特性，勾進既有系統，如耐高溫 / 夜視 / 精煉效率），
  **不做完整技能樹**——否則美術量與數值平衡會爆。
- 誠實提醒（單人開發）：種族在玩法 / 資料上便宜，但**美術昂貴**（每種族整套動畫 / 裝備
  sprite），所以種族主要做「外觀 + 小特性」。

## 6. 帳號身份模型 + 登入

> 結論：**身份模型第一天就要做對（難回頭）；Google OAuth 當第一個登入方式（便宜、UX 好）。**

- 真正難回頭改的，是**玩家身份模型**——存檔、地盤所有權、session 全都綁在「使用者 ID」上。
  因此第一天就把它設計成 **provider 無關**：內部一個 user ID，外部登入方式只是「連結」上去。
- **Google OAuth（用 Google 登入）**對網頁遊戲很適合（免密碼、免註冊摩擦），所以第一天就把它
  當第一個登入 provider 實作，但底層留好介面，之後 Discord / Apple / email 都能插上去，不重寫。
- 換句話說：「後面再加不好加」的不是 Google 本身，而是身份模型。模型對了，Google provider
  任何時候加都便宜。

## 7. 技術選型

### 7.1 前端

- **HTML5**：世界用 **Phaser 3 / PixiJS**（2D 俯視）；競速賽道未來可用 Three.js 偽 3D。
  瀏覽器跑、觸控 + 鍵鼠、免安裝、跨平台。
- **美術**：**像素風**（pixel art 16 / 32 bit）。現成素材包多，對單人最友善，風格壓得簡單一致。

### 7.2 後端（綠地重來，已鎖定 Rust + Postgres）

> 現有 repo 的舊殼只是歷史，不沿用；後端從零開始，無歷史包袱。

- **非同步執行緒 + Web 框架**：tokio + axum（axum 內建 WebSocket 支援）。
- **WebSocket**：axum ws（或 tokio-tungstenite）。
- **Postgres**：sqlx（非同步、編譯期檢查 SQL；比 diesel 更適合 async / 單人）。
- **Redis**：redis-rs / fred（即時熱資料、presence、session）。
- **序列化**：serde + serde_json（MVP）；之後可換 bincode / rmp-serde 走二進位省頻寬。
- **遊戲迴圈**：固定 tick 的權威伺服器（約 10~20 Hz），對同一 zone 廣播狀態快照 / 差異。
- **實體管理**：MVP 可簡單寫；要進階可用 bevy_ecs 當函式庫。
- **架構**：權威伺服器（客戶端送輸入、伺服器模擬、廣播狀態）；以 zone / 星球做空間切分供未來擴展。

## 8. 北極星 vs 可出貨 MVP（最重要的紀律）

- **北極星願景**：多星球、星際、海陸空外太空的乙太宇宙。**這是長期方向，不是第一版。**
- **殘酷事實**：多星球星際 MMO ≈ EVE / No Man's Sky / Star Citizen 量級，需要大團隊數年。
  一人 + AI 做得出的是「**朝它生長的遊戲**」，不是「完成的星際宇宙」。一開始就做星際 =
  永遠出不了貨。
- **對策**：MVP 只做「**一顆星球、一座城鎮**」的療癒小切片；架構為星際預留（地圖分區、
  星球 = zone、飛船港口 = 未來入口）。飛船停在母星港口，玩家看得到未來，但先把腳邊做好玩。

## 9. Phase 0 MVP（精確範圍）

### 要做

- **provider 無關的帳號身份模型** + **Google OAuth** 當第一個登入方式。
- 一顆星球地表的**一張城鎮 tilemap**。
- 玩家角色 + WASD / 觸控移動。
- **一個起源（地球人 / 繼承農莊）**，但創角 / 起源系統做成可插拔。
- 即時看到其他玩家移動 + 文字聊天（WebSocket）。
- 個人農地：耕地、種 1~2 種作物、澆水、隨日夜收成。
- 伺服器驅動的日夜循環。
- 持久化：角色位置 / 背包 / 農地存 Postgres，線上狀態 / presence 存 Redis。

### 不做（留待後續 Phase）

戰鬥、載具 / 卡丁車、自動化機台、職業系統、多星球、飛船。

### 目標

先攻克最難的「**網頁即時多人 + 持久化**」，配最療癒的最小內容。

## 10. 六款風格如何咬合（不是並列，是互相餵養）

種田（星露谷 / 牧場）、自動化（缺氧）、採集建造（麥塊）、職業社交打怪（RO）、
開車競速（GTA / 極速）——全部透過第 4 章的乙太燃料經濟互相餵養：

農場種植 / 提煉燃料與零件 → 燃料供應自動化機台 + 載具 + 飛船 → 競速 / 探索消耗燃料、
產出零件獎金 → 回頭餵農莊工坊。

## 11. 分階段路線（每階段可玩、可喊停）

| 階段 | 主題 | 內容 |
|---|---|---|
| **Phase 0** | MVP：網頁即時多人 + 持久化 | 見第 9 章 |
| **Phase 1** | 開車與採集（GTA + 麥塊） | 地表 2D 開車、野外採集、合成台、工具科技樹、佈置地盤 |
| **Phase 2** | 深度與自動化（牧場 + 缺氧） | 動物、加工機台、灑水 / 自動化、水電（蒸汽壓力 / 管路） |
| **Phase 3** | 社交與職業（RO） | 職業 / 技能、戰鬥、組隊、公會、玩家交易、市集 |
| **Phase 4** | 卡丁車競速（極速 / 地平線） | 偽 3D 卡丁賽道、改裝、計時 / PvP、賽事經濟 |
| **Phase 5+** | 星際擴張（北極星） | 飛船航行、第二顆星球、星際旅行 / 探索 / 競速 |

## 12. 誠實警告與風險

1. **真瓶頸是美術素材 / 內容量 / 數值平衡，不是寫程式** → 用現成 tileset 或 AI 輔助美術。
2. **「2D 俯視世界」+「偽 3D 賽道」+「星際飛行」是不同渲染管線** → 要接受它們是分開的模式。
3. **第一天就多人連線讓 MVP 較難**（同步 / 權威 / 持久化），但符合願景與 RO 靈魂，值得。
4. **範圍蔓延是頭號殺手**；星際願景更要嚴守「先做一顆星球一小塊」。
