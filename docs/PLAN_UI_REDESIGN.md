# HUD / 介面重設計 — 從「一面牆的圖示」到「看得懂、不佔螢幕」

> 願景來源：**玩家 2026-06-11 親口需求 + 實機截圖**——「一堆圖案我看不懂；全加文字又會占滿；
> 是不是做浮動視窗、像 RO 那樣？打開一堆項但不知道什麼是什麼」。設計：AI 規劃(玩家已指方向)。

## 問題（實機截圖診斷）

手機實機上，HUD 下方是**一整面牆約 20+ 個 emoji 按鈕**(⚔️🔨🔮🐾🎣🐔🌾✨📦🔨📜🗺️📦🏅🎭⚜️🏆📅…)，
佔掉近半個畫面，且：
- **看不懂**：純 emoji、無文字標籤，玩家得逐一猜每顆是什麼。
- **佔螢幕**：全加文字會塞爆；但全是圖又難辨識。
- **不知道什麼是什麼**：所有系統面板平鋪同一層，沒有分類、沒有層次。

根因：**每加一個功能就塞一顆常駐 dock 按鈕**累積出來的 UI 肥大(資訊架構問題，不是美術問題)。

## 設計方向（RO 風 + 手機優先）

1. **常駐欄瘦身**：螢幕上永遠只留**必要的**——背包、地圖、角色、`☰ 選單`，加關鍵狀態(乙太/血/等級/職業)。
   其餘系統入口**全部收進選單**，不再平鋪佔畫面。
2. **`☰ 選單` 開「分類 + 有文字標籤」的面板**：每個系統是**圖示 ＋ 文字**(如「⚔️ 戰鬥」「🌾 農作」)，
   依功能分組：**戰鬥 / 生產採集 / 角色養成 / 社群 / 活動其他**。玩家一眼看懂、找得到。
3. **點開＝對應視窗**：像 RO 一樣**可拖、可關、可疊**的浮動視窗；同時最多開幾個自己排版。
   小螢幕自動改為單一全寬面板，避免擁擠。
4. **消除神祕圖示**：長按/hover 顯示提示(tooltip)；面板內一律帶文字。

## 鐵律

- **不破壞既有功能**：所有現有入口都還在，只是**重新組織**到分類選單/視窗裡，行為不變。
- **手機優先**：大點擊區、少常駐元素、可收合；先確保順、別為動畫犧牲流暢。
- **i18n**：所有標籤都是面向玩家字串，**集中、可翻譯，別寫死**(沿用專案 i18n 方向)。
- **原創**：版面/圖示自己做，不抄外部遊戲 UI 素材(可吸收 RO 等的互動概念，寫原創實作)。

## 漸進順序（一次一塊，玩家每步都有感）

1. **把現有 dock 收進一個分類選單**：常駐欄瘦到必要按鈕；`☰ 選單`展開「圖示＋文字＋分組」的列表。
   ——這一步就解掉「看不懂 + 佔螢幕」大半，先做、最有感。
2. **面板改成可拖/可關的浮動視窗**(RO 風)；小螢幕退化為全寬單面板。
3. **細節打磨**：tooltip、記住上次開的視窗位置、常用功能可釘選回常駐欄(玩家自訂)。

## 優先級

**高**——玩家現在實機就卡在這、影響每一次遊玩。建議排在視覺/3D 之前先落地(投入小、回報大)。
