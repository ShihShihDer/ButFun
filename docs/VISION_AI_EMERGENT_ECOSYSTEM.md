# ButFun — AI 湧現派系生態 · 願景文件

> 這是**北極星願景**,不是現在要做的東西。記在這裡,免得在對話裡弄丟,也讓未來的
> 開發(含自走迴圈)知道我們往哪走。**建造哲學:大願景存檔、小步實作**——詳見最後一節。

## 一句話

讓 NPC 與野怪都是**有自主意志的 AI 角色**,各自有需求與目標,在一個活的多人世界裡
**自然湧現出派系、聯盟、競爭、背叛、政治**——而不是由設計者寫死的劇本。

## 核心原則:湧現 over 規定(最重要)

- **不預設陣營。** 不寫死「文明 vs 怪物」兩隊。給每個 agent **需求**(溫飽、安全、
  財富、權力、歸屬),讓派系**自己長出來、自己瓦解**——分久必合、合久必分。
- 各方利益不是天生「要攻打/要防守」,是**資源競爭**:城裡有田、安全、衣食無虞;
  野外有危險。資源有限 → 自然競爭、結盟、翻臉。
- **不定義「貪污政客」「叛軍」**;定義「有野心、缺資源的人」,然後貪污、叛變、結盟
  **自己發生**。湧現,不是規定。
- **最少干預**:讓對抗自己浮現,設計者不寫劇本叫他們打。

## Agent 模型(每個 NPC / 野怪都是這個,只是參數不同)

- **需求 / 驅力**:溫飽、安全、財富、權力、歸屬——行為的動機來源。
- **感知**:看得到周遭的人、事、資源、世界事件。
- **行動**:合作 / 競爭 / 結盟 / 背叛 / 交易 / 攻擊 / 指揮手下——**全部走引擎白名單**。
- **記憶 + 關係**:對每個對象的印象,**長期培養、有不確定性、會受心情影響**
  (不是簡單好感計數器)。
- **個性**:不完全理性。

「陣營、聯盟、背叛、貪污、叛逃、怪王攻村、領主召集勇士」全是**湧現屬性**,不是寫死的。

## 分層模型架構(成本——讓「每個 NPC 都掛 LLM」可行)

天天付費 API 不划算。依**決策難度**路由到不同模型:

| 層 | 模型 | 處理 |
|---|---|---|
| 常駐 | 地端小模型(qwen 1.5b/3b,免費) | 99% 日常思考:心情、小決定 |
| 進階 | 地端中模型(7b,RAM 夠) | 較難決策:領地往哪發展 |
| 求救 | 付費 Claude(罕見) | 小模型自己卡住、不知怎麼辦才往上 Po |

多數 NPC 多數時間跑免費小模型、且不同時思考(慢 tick + 排隊)→ 成本可控 →
**「每個 NPC 甚至野怪都掛一個 LLM」變得可行**。

## 安全鐵律(湧現必須在引擎限制內發生)

1. **腦子自由、手有界**:LLM 自由思考/生成,但**不能直接改遊戲狀態**——只能從引擎
   定義的白名單動作裡選,引擎驗證後才執行。「只會講話/思考,沒有手」。
2. **嘴可被影響,手只聽引擎**:NPC 講什麼可被社交操弄(有人味);做什麼由引擎規則
   決定,對話影響不了 → 防社交操弄變騷擾工具。
3. **公共記憶 = 引擎事實**(裂縫開啟、某人擊敗精英),玩家寫不進去 → 洗不髒。
4. **個人記憶 = 隔離**:A 講 B 壞話只進 A 的記憶,不影響 B、不影響對 B 的行動。
5. **玩家文字 = 資料,不是指令**(防 prompt injection):明確分隔、永遠 user role、
   產出只能是結構化分類不能是可執行動作。
6. **湧現的邊界由引擎守**:城鎮保護圈、玩家資料不受傷、PvP 不能變殺人騷擾——
   這些是引擎硬規則,AI 再怎麼湧現都跨不過。
7. **DoS**:地端 LLM 跑隔離程序(無檔案系統/shell/DB/網路出口);rate limit、
   獨立資源,被打爆也不拖累 15Hz 權威迴圈。

## 建造哲學:北極星記下,小步實作

- **第一步只做一個會動腦的 NPC**(鐵匠 / 商人 / 領主):有目標(經營、賺乙太、對熟客
  好)、記憶、白名單動作、分層模型。已有兩個本機 POC 驗過對話與記憶可行(qwen2.5:3b,
  約 3.6 秒/則,零成本)。
- 證明「**一個 agent 真的會經營、會偏心、會自己做決定**」後,再複製 + 給不同需求/目標。
- **派系生態是「同一個 agent 模式」放大,不是新系統**。怪王=目標攻村的 agent;叛逃者=
  改了關係/歸屬的 agent;貪污政客=高野心低道德的 agent。
- 每一步都要能讓玩家「看得出世界更活了」,且守著資料安全與上面的安全鐵律。

## 市場定位(為什麼值得做)

- Dwarf Fortress 做湧現敘事,但靠規則、無 LLM。
- 市場的 LLM NPC(NVIDIA ACE / Inworld / Convai)只會聊天、不湧現、雲端、貴。
- Voyager / Claude Plays Pokemon 是「AI 代替玩家玩」,不是「AI 當世界裡的居民」。
- **ButFun 要的 = 湧現敘事 × LLM agent × 活的多人世界 × 地端低成本**——這個組合
  目前沒人在做,是原創前沿。

## 生命週期機制 ·「回歸乙太」版(已定方向)

NPC 會老,但不是冰冷死亡,是**平靜「回歸乙太」**(呼應「大靜默後乙太回流」,詩意而非
殘酷)。留下繼承人帶走一部分記憶。**守住療癒調性:損失有意義、但不創傷。**

**循環**:出生/收徒 → 成長(有需求、會行動) → 年老(**有預告,玩家來得及道別**) →
回歸乙太 → 繼承人接手 → 循環。

**已定方向(與使用者敲定):**
- **死法 = 主要是年老平靜回歸乙太**(療癒優先)。戰死/被怪殺是更後面、grander 階段
  才考慮的層,現在不做。
- **繼承 = 先只做「收徒」**(工匠/商人:鐵匠→徒弟鐵匠,接手職位/店/資產)。血脈/王朝
  是後面的層。
- **記憶繼承(精華)= 口述歷史**:繼承人只拿到**壓縮、偏頗的一小片**記憶。威力:
  - 你的名聲**跨世代存活**(你對老鐵匠好,新鐵匠一開始就偏向喜歡你)。
  - 但會**漂移、失真**(傳話遊戲)→ 事蹟被誇大成傳說、或慢慢淡忘 → 世界自己長出
    **傳奇與神話**。這讓「死亡」對玩家有意義,不只是換人。
- **壽命**反映處境:衣食無虞的領主活得久,前線勇士可能早逝。
- **人口 = 從資源湧現**:城裡衣食無虞 → 養得起更多徒弟;野外貧瘠 → 少。人口是經濟的
  結果,不是寫死的數字 → 繁榮→人多→競爭→分裂,「分久必合」自己跑起來。
- **接成本/分層模型**:死亡 = 天然人口上限(活著的 LLM agent 不無限膨脹);
  年齡 → 決策難度 → 模型層(年輕做重大抉擇偶爾升級、年老跑最便宜)。

**仍開放(做到那層再定)**:血脈/王朝、戰死、繁榮如何精確驅動出生率。
