# 多渲染器架構 — 一個世界，多種視角

> 願景來源：**玩家 2026-06-11 親口需求**——「渲染可選(高/低畫質)，高畫質用猛的引擎；
> 有點想走 2.5D/3D，能不能共存？一個世界多種視角？」**並授權：框架/設計自由發揮。**
> 結構/引擎選型/鐵律：AI 規劃。

## 為什麼共存是乾淨可行的（核心）

ButFun 是**伺服器權威 + 資料驅動**：伺服器送出的世界是**座標 + 型別**(玩家在 (x,y)、
地形＝wasm 的 tile/biome 碼、敵人/NPC/物件的位置型別)，**不是畫好的像素**。
現在的 Canvas 只是「拿這份資料來畫」的**其中一個**消費者。

再加一個 3D 渲染器＝**讀同一份世界狀態、用另一種方式畫**：同樣 (x,y) 攤成 3D 地面、
地形碼決定高度/材質、實體用立牌(billboard)或模型、鏡頭可俯視可斜角。
**切換視角不改玩法**——碰撞、移動、互動永遠是伺服器那套 2D 邏輯。這就是真正的
「一個世界、多種視角」，2D 與 2.5D/3D **可共存、可選**。

## 建議：2.5D 當高畫質 tier，引擎用 Three.js

- **2.5D 是甜蜜點**：保留俯視療癒框架與辨識度，加真實高度/光影/陰影——又美又忠於玩法、手機高階可跑。
  全自由 3D 鏡頭可後續當一個選項。
- **Three.js**：web 3D 標準、手機可、生態大。**玩家已授權引入框架**——走正常 review 即可，不需額外人工關卡。

## 四條鐵律（不可破）

1. **玩法永遠 2D 權威**：3D 純視覺。碰撞/移動/互動/wasm 地形邏輯**完全不變**，切視角不影響公平與手感。
2. **低畫質永遠是 Canvas 2D 安全保底**：3D 是 opt-in；弱機/不支援 WebGL 自動退回 Canvas。**絕不為漂亮犧牲順**。
3. **效能/手機**：分塊載入地形、視野外剔除、billboard、draw-call 批次、LOD、品質分級。
4. **原創美術/模型**：自己做或程序生成，**絕不抄外部遊戲資產**。

## 漸進建造順序（一次一塊，地基先行）

1. **渲染器抽象層**(零風險、零新相依、零視覺變化)：把「遊戲狀態 → 畫」抽成 `Renderer` 介面，
   現有 Canvas 收斂成 `CanvasRenderer2D`。**先做這步**，它讓多渲染器成為可能，且不改任何畫面。
2. **最小 3D 渲染器**(Three.js，藏在設定開關後)：平地 + billboard 實體 + 俯視鏡頭對齊 2D。
   設定可切 2D/3D。**目標：證明「同一份世界狀態、兩種視角」真的共存。**
3. **加 2.5D 深度**：斜角鏡頭選項、地形高度起伏、光影與陰影。
4. **漸進豐富**：更好的模型/立牌、環境光、水面、天氣、生態氛圍。
5. **畫質分級 + 自動偵測**：低＝Canvas 2D、高＝2.5D/3D；玩家可切、可持久化、依裝置能力自動預設。

## 範圍誠實話

這是目前接過**最大的一條線**，會跨很多輪慢慢長。風險可控：**步驟 1 安全可先做**、
**Canvas 2D 全程是預設保底**。與 `docs/PLAN_VISUAL_POLISH.md`(Canvas 精緻化＝低畫質 tier)、
`docs/PLAN_UI_REDESIGN.md`(HUD 重設計)並行不衝突。**優先級上，HUD 重設計建議先做**(玩家當下就卡著)。
