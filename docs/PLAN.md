## 🎯 現在主攻:給「收成/乙太」一個去處——先 land 合成,解玩家最痛的「填滿即停滯」

採集→背包(PR#18)、改名(PR#19)、大世界(PR#20)、per-player 農地(PR#12)已上線。
本輪 30 則玩家回饋**高度收斂在同一句**:**「農地全填滿就沒事做、收成/乙太沒去處、世界太靜」**
(居民-5「全填滿不知做什麼、停滯感」反覆出現、居民-7「旁邊空間用完了想擴地」、居民-1「想要設備工具/
可預測的次級循環」、居民-10「環境變化低、期待活動點位/會動的東西」)。

這把優先序講清楚了:**玩家當下最痛的是「產出沒有去處」**(經濟迴圈半邊空轉)——GDD 紀律白紙黑字
「乙太有產出也要有去處」正是此事,**它比戰鬥更貼近玩家此刻實際喊的痛**。對應兩個去處:
- **素材的去處 = 合成**(背包木/石 → 工具),不動廣播 shape、風險最低 → **本輪第一優先 land**。
- **乙太的去處 = 擴地**(收成換更多地),直接回應居民-5/7「填滿停滯/想擴地」。
戰鬥(解「太靜/想要會動的東西」)仍是對的方向、維持推進,但它要動廣播 shape(加 `enemies`)=
本 repo 一直 land 不了的那種,**別再讓整條鏈卡它**;玩家也沒直接喊打怪,喊的是「沒事做」。

### 給維護者的優先序(一次接一條,別並行撞 main)
1. **先 land 合成(1-C)**:接 `crafting.rs`(配方已建,additive 摘檔同採集法)。背包已隨快照廣播,
   合成只是「client 送 `Craft` → 伺服器查配方扣素材 → 背包走**既有**快照」,前端只多一個面板=**零契約變更**。
2. **再 land 工具用途(1-D)**:身上有鎬子採礦更快(`tools.rs` 已建)。閉合「素材→合成→更快採」第一個正回饋圈。
3. **擴地(0-G-O2,乙太買地)**:玩家直接喊的去處。per-player fields 已是 Vec 廣播,**能複用既有 shape
   就低風險先上**(同玩家認領相鄰新地塊);需動 shape 才走 PR。
4. **戰鬥(1-F)**:維持方向——從 main 開乾淨枝、摘 `combat.rs`/`enemy_field.rs`/`vitals.rs`(additive、
   別碰卡衝突的 PR#17 對既有檔的過時編輯)、加 `enemies` 快照欄位(前端 `|| []`)、**開 PR 給審**。

## 接下來 1-3 個切片(由上往下,每個附「驗收=玩家能做到什麼」)

1. **合成 MVP(1-C)— 最低風險、先 land**:木×3+石×2 → 鎬子,不動廣播 shape。
   **驗收 = 玩家把背包素材合成出鎬子、收成終於有去處**(經濟迴圈起步,直接解「填滿即停滯」)。
2. **工具效用(1-D)— 閉環**:有鎬子採礦更快、沒有就慢。先只做鎬子×採礦一條。
   **驗收 = 合成出的鎬子真的讓採集變快**,第一個完整正回饋圈。
3. **擴地 MVP(0-G-O2)— 乙太的去處**:收成的乙太換一塊相鄰新地塊。薄:固定價、固定下一格、無市場。
   **驗收 = 玩家花乙太多拿一塊地、填滿後不再停滯**,直接回應居民-5/7。
