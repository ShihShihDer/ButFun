## 🎯 現在主攻:把乙太的「去處」做出來(收口經濟迴圈)

玩家收成只讓乙太數字一直變大,**沒有任何地方花** → 沒有目標、沒有成長、沒有「再玩一下」的理由。
這是「沒啥好玩」的頭號病根。per-player 農地與位置+乙太持久化都已 merge 進 main,地基齊了——
現在把**乙太的消耗去處**做出來,玩家就有「存乙太 → 換到看得見的東西」的動機,種田迴圈才閉環。
對齊 BACKLOG **Phase 0-G-O2**,仍在 Phase 0、不跳級。

## 接下來 1–3 個切片(由上往下)

1. **乙太花得出去:擴自己的地(單一最小消耗點)**
   驗收 = 玩家能用收成的乙太「把自己那塊農地擴大一格」,乙太數字**變少**、可耕格**變多**,
   且**擴張跨重啟還在**。先只做這一個消耗點,不必一次做完商店。
   選「擴地」而非「買種子」當第一個 sink:擴地直接給「種田→更多種田容量」的成長感與擁有感,
   且不必先把種子做成稀缺資源(買種子要先有種子限量,複雜度更高,延後)。
   ⚠️ **前置**:乙太已持久化、農地/地塊狀態卻還沒——若擴出來的格子重啟就消失(乙太卻真扣了),
   是「付了錢東西不見」的真體驗坑。所以**農地+地塊持久化要先落地**(見 backend),sink 才站得住。
2. **作物有選擇:第二種作物**
   驗收 = 玩家能種出兩種不同作物(成長時間 / 乙太回報不同),種田開始有「要種哪個」的決策。
3. **上手第一分鐘的鉤子**
   驗收 = 新登入玩家進場後,被清楚引導完成「站到自己的地 → 種下第一顆 → 收成 → 看到 +乙太」一輪,
   不靠猜。讓第一次就嘗到完整迴圈的甜頭。

## 給各線的具體指示

- **backend(本輪關鍵路徑,請先回到這條)**:把**農地+地塊登記持久化進 Postgres**(`Field` 的
  tiles/crop 狀態、`PlotRegistry` 的 user_id→序號表),沿用 0-E 既有路數(`db.rs`/`migrations` 已建好
  players 表的模板、記憶體 cache + 背景 flush、`from_tiles`/`from_saved` 載入時驗證、無 `DATABASE_URL`
  退回記憶體)。**這是切片 1 sink 站得住的前置**——位置+乙太已穿磁碟、農地卻沒,先補平。接著支援
  擴地動作的權威處理(扣乙太+擴格,跨重啟一致)。
  ⚠️ **別放哪**:別再往 Phase 1 採集節點接線(`node_scatter.rs`/`gather*` 接線,如 open PR #13)推——
  那是跳級,且把該清的農地持久化 blocker 晾著,等於拖住整個主攻。純邏輯地基已夠多,先把這塊接穿磁碟。
- **feature(玩法)**:扛切片 1「擴自己的地」——backend 持久化補平後即可接上消耗動作。小步、集中新檔。
- **frontend**:把「種田迴圈」做得**一眼讀懂** + 切片 3 上手引導(進場→種→收→+乙太飄字);
  擴地要有清楚的「花 N 乙太 → 多一格」回饋。減少再投資邊角無障礙/動效微調——那些已很完整。
- **feedback**:挑玩家建議裡「讓核心迴圈更清楚」的小事優先(提示、可讀性),其餘照舊。
  (本輪 `data/suggestions.jsonl` 無新進回饋。)
