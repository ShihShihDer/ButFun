## 🎯 現在主攻:武器/裝備 — 玩法鏈最後缺口,additive 可獨立 land

玩法鏈是 **採集→背包→裝備/物品用途→戰鬥**。戰鬥已全上線(自動鎖定/反擊/扣血/打趴/
掉落進背包),且 **PR #27「活起來的世界」剛 land——敵人會巡邏、主動追擊你**。但中間
「**裝備/武器**」這環被跳過:傷害是寫死常數 `PLAYER_ATTACK_POWER`,**玩家對戰鬥零選擇**。
敵人現在會追上來打你、你卻只能徒手挨打反擊——**無 agency 比以前更刺眼**。補上武器=閉合
「採集→合成→戰鬥變強」正回饋圈,正是總監點名的 Phase 1 主攻,不是跳級。

**為什麼最快見效**:武器走**已驗證的 1-D 工具 additive 接法**——採集那行(`ws.rs`)早已
`tools::gather_speed_multiplier(&inv)` 查表;戰鬥那行 `game.rs:174` 還是寫死
`PLAYER_ATTACK_POWER`。武器只要:`ItemKind::Weapon` 變體 + `crafting.rs` 一條武器配方 +
`combat::weapon_power(inv)` 查表 + **game.rs 一行**把常數換成查表。**無新 protocol、騎現有
craft+背包廣播 shape** → 比擴地(需全新 `ClientMsg::BuyExpansion`、卡多輪)快得多、風險低得多。

> **本輪更正(重要)**:上份 PLAN 叫維護者「摘 feature 線備好的 Weapon 件」——**但 feature
> 根本沒做**:`auto/feature` 最近 commit 全是 1-E/1-F 戰鬥/騎乘測試,**沒有 `ItemKind::Weapon`、
> `weapon_power`、武器配方**。所以別再去 feature 找不存在的零件。武器件極小且純 additive、
> 無新 protocol——**這輪由維護者連寫帶接一次做掉最快、最安全**(連寫帶接也不碰廣播 shape)。

### 給維護者的接線優先序(一次一條,別並行撞 main)
1. **武器(裝備 MVP)— 連寫帶接**:`inventory.rs` 加 `ItemKind::Weapon`、`crafting.rs` 一條
   武器配方(如 鐵×?+乙太×? → 武器)、`combat::weapon_power(&inv)` 查表(鏡像
   `tools::gather_speed_multiplier`:有武器回高攻擊力、沒有回 `PLAYER_ATTACK_POWER` 徒手值),
   `game.rs:174` 把常數換成查表。**additive、無新 protocol、最低風險。**
2. **擴地(0-G-O2)**:`protocol` 加 `ClientMsg::BuyExpansion`、`ws.rs` 收下、`game.rs` 查
   `economy::expansion_cost` 扣乙太、給該玩家多一塊相鄰地塊。前端面板已備。複用既有 per-player
   `fields` Vec 廣播(新地塊=多一個 Field)最低風險;真要動 shape 才走 PR。**架構級,排第二。**
3. **載具(1-E)上下車**:純邏輯已在 `auto/feature`(騎乘迴圈測試 `265bfd7` 等),維護者
   **先摘回 main** 再接上下車意圖。給世界移動爽感+探索。

## 接下來 1-3 個切片(每個附「驗收=玩家能做到什麼」)

1. **武器 MVP — 戰鬥 agency(additive,先 land)**:合成武器,身上有武器打怪更痛/更快。
   **驗收 = 合成出的武器真的讓打怪明顯變強**,戰鬥第一次有玩家選擇(像 1-D 鎬子讓採集變快)。
2. **擴地 MVP(0-G-O2)— 乙太的去處**:固定下一格、逐格漲價、無市場。
   **驗收 = 玩家花乙太多拿一塊相鄰地塊**,收成/打怪的乙太終於有去處(GDD「產出要有去處」閉合)。
3. **載具 MVP(1-E)— 移動爽感**:走近按鍵上車、3 倍速+輕微甩尾、下車回走路。先一台車一人騎。
   **驗收 = 上車明顯比走快、能跑去採集圈探索、下車回原樣**。

## 給各線的具體指示

- **feature**:上份指示的武器件**沒做出來**——若這輪要動,就**只交武器那三個純函式**
  (`ItemKind::Weapon`+一條配方+`weapon_power` 查表,附測試,鏡像 `tools` 寫法),**無 IO、
  不動 ws/game/廣播 shape**;否則別動。**停堆 1-E/1-F 戰鬥/騎乘的不變式遍歷測試**(已飽和)、
  **別開 ws/game/protocol 接線 PR**(接線走維護者)。
- **frontend**:HUD 星露谷 dock 化已 land,**非接線可玩面已飽和、停手等接線**。別再補小地圖
  圖例/多快捷鍵/微調既有面板——邊際趨零。武器上線後做「手上武器」HUD 小圖示(預備一件、做完
  就停);擴地後補「買地確認/新地塊視覺」;載具後補「上下車提示」。新快照欄位一律 `|| []` 防呆。
- **backend**:0-E 五核心 store+建議箱 PG+改名 live 廣播全 land,**基本待命**。武器是 additive
  (背包已持久化+廣播,零新持久化);擴地/載具大接線由維護者一次一條做。**別搶、別開接線 PR**,
  真要動就挑能獨立 land 的小事(純函式測試補強、非接線內部清理)。
- **feedback**:戰鬥/合成/操作說明提示已足夠,**停手**。等武器上線補「怎麼合武器/武器更強」、
  擴地補「乙太怎麼買地」、載具補「怎麼上車」。在那之前別微打磨既有提示。
