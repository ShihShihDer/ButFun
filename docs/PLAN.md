## 接下來的切片(由上往下、薄切片快速連發)

1. **薄背包:採集 → 進背包 → 看得見**(現在這條)
   驗收 = 玩家在世界裡採集到一個物品 → 出現在可開關的「背包」面板,跨重連還在。
   只做這條最小迴圈,**別**一次把合成/裝備/堆疊全做進來。
2. **物品有用途**:採到的東西能做點什麼(賣乙太 / 餵作物 / 當材料),讓背包不只是清單。
3. **戰鬥 MVP(自動打怪)**:世界出現可打的目標,角色能**自動攻擊**附近敵人、掉落物進背包。
   先做最薄的一條:有敵人、會自動打、掉東西。裝備/技能/數值成長隨後一條條補。
4. **(並行,別擋路)經濟 sink + 持久化**:乙太擴地、農地/背包/戰鬥狀態存 Postgres——
   能讓功能先上、持久化隨後補的就先上。

## 給各線的具體指示(分工清楚 = 少撞檔)

- **backend(主力)**:把 `inventory.rs`/`gather.rs` 接進權威伺服器(Player/AppState 帶背包、
  `ws.rs` 加採集動作、`protocol.rs` 快照帶背包)。接著接戰鬥邏輯。**動 live 廣播 shape → 走 PR、
  不自走 merge、別在共用工作樹留半成品、前後端同版上線**(比照 per-player 慘痛經驗)。持久化並行補。
- **frontend**:`web/game.js` 做**背包面板** + 採集/戰鬥回饋。新快照欄位一律 `|| []` 防呆(沒接好不崩)。
  **打磨暫緩**——無障礙/動效/安全區已夠完整,力氣全給新玩法的顯示。
- **feature**:做玩法「內容」——可採集的東西、採集節點怎麼擺、敵人種類/掉落,盡量寫新檔,
  別跟 backend 搶同一段 ws 接線。
- **feedback**:挑「讓新玩法更好懂」的小提示優先。
