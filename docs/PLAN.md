## 🎯 現在主攻:把乙太的「去處」做出來(收口經濟迴圈)

目前玩家收成只會讓乙太數字一直變大,**沒有任何地方花** → 沒有目標、沒有成長、沒有「再玩一下」的理由。
這是「沒啥好玩」的頭號病根。把**乙太的消耗去處**做出來,玩家就有了「存乙太 → 換到東西」的動機,
整個種田迴圈才閉環。對齊 BACKLOG **Phase 0-G-O2**(用乙太擴張/購買),仍在 Phase 0、不跳級。

## 接下來 1–3 個切片(由上往下)

1. **乙太花得出去:擴自己的地 / 買種子**
   驗收 = 玩家能用收成的乙太「擴大自己那塊農地一格」或「買一包種子」,乙太數字會**變少**,
   且擴張/購買跨重啟還在(接 Postgres)。先做一個最小的消耗點即可,不必一次做完商店。
2. **作物有選擇:第二種作物**
   驗收 = 玩家能種出兩種不同的作物(成長時間 / 乙太回報不同),種田開始有「要種哪個」的決策。
3. **上手第一分鐘的鉤子**
   驗收 = 新登入玩家進場後,被清楚引導完成「站到自己的地 → 種下第一顆 → 收成 → 看到 +乙太」一輪,
   不靠猜。讓第一次就嘗到完整迴圈的甜頭。

## 給各線的具體指示

- **feature(玩法,主力)**:扛切片 1「乙太花得出去」——這是現在最重要的玩法。小步、集中新檔。
- **backend**:支援切片 1 的伺服器邏輯(乙太扣除/擴地的權威處理 + 跨重啟持久化),把農地/乙太的
  持久化補齊(目前位置+乙太已串 Postgres,農地狀態還沒)。
- **frontend**:把「種田迴圈」做得**一眼讀懂** + 切片 3 的上手引導(進場→種→收→+乙太飄字)。
  减少再投資在邊角無障礙/動效微調——那些已經很完整了。
- **feedback**:挑玩家建議裡「讓核心迴圈更清楚」的小事優先(提示、可讀性),其餘照舊。

## 暫緩(現在先別做)

- 無止盡的打磨(更多 safe-area / 動效 / 報讀器微調)——已足夠完整,邊際效益低了。
- Phase 1.5 捏角色、行銷/獲客、飛船/多星球/競速——**核心種田迴圈夠好玩之前都先擱著**。
- 任何跳級到 Phase 1 之後的大功能,除非 feature 線的當前切片已完成。
