## ✅ 採集→背包 已上線(PR #18,維護者接線)

## 🔑 關鍵情報:瓶頸是「接線」不是「造零件」

自走 lane **很會造零件、卻接不動線**(架構級接線會互撞 main、整合器只做乾淨合 → 永遠合不進去)。
**一大票玩法純邏輯早就建好、沒接線**,躺在 `auto/feature` 分支上:
**戰鬥(1-F 敵人/自動鎖定/生命值/威脅=自動打怪)、合成(1-C)、工具(1-D)、載具(1-E)、
乙太擴地 sink(0-G-O2)、改名 API**。零件齊了——接線一律由**維護者一次一條薄切片**做。

## 🎯 下一條(給維護者接線的優先序)

1. **改名(高優先、最小)**:`UserStore` 改名後端半已在 `auto/feature`;接 `PATCH /api/profile` +
   前端設定面板 + live 廣播。驗收 = 改名後自他即時看到、重連仍新名。玩家多次點名要、且最薄。
2. **戰鬥 MVP / 自動打怪(總監要)**:1-F 的敵人/自動鎖定/生命值/威脅純邏輯已建好,接進 `ws.rs`/
   `game.rs`/快照 + 前端畫敵人/血條。驗收 = 世界有敵人、走近自動打、掉落進背包。**零件已齊,照採集→背包同法接。**
3. **物品有用途**:採到的木/石能合成(1-C 配方已建)或賣乙太,讓背包不只是清單。
4. **持久化補平**(並行、別擋路):背包/農地/節點存 Postgres(都有 `from_raw`/`is_loadable` 載入防線)。
