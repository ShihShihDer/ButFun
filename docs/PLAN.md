## 🎯 現在主攻:合成(1-C)接線——玩法鏈裡唯一還沒接通的那一環

玩法鏈三來源都在灌背包(採集 PR#18、打怪掉落 PR#21、農地收成 0-G),素材堆得快、
卻**沒有去處**——乙太只進不出、木石純囤積。GDD 白紙黑字「乙太有產出也要有去處」此刻是**最大缺口**。
去處 = 合成(背包木/石/廢料 → 工具)。

**而且自上輪起,合成的周邊全部就緒,只差最後一根接線:**
- `crafting.rs` 配方已在 main、`tools.rs`(1-D 鎬子效用)也已落地——**純邏輯齊了**。
- **背包已接 PG(PR#7603509)且早已隨快照廣播**(`protocol` 的 `players[].inventory`)。
- **前端合成台 + 背包面板已建好**(b625f47、dac7d54)、HUD 限高內捲(23ee52c)——**UI 備好了**。

所以合成 MVP 現在是**零契約變更、風險最低、最快見效**的一筆:client 送 `Craft` →
伺服器查配方扣背包素材加道具 → 背包走**既有**快照回前端。**唯一缺的就是 `ws.rs`/`game.rs`/`protocol`
那個 `Craft` handler**(目前 `src/` 只有 `mod crafting;`,完全沒被呼叫)。→ **本輪維護者第一優先。**

### 給維護者的接線優先序(一次一條,別並行撞 main)
1. **先 land 合成(1-C)**:`protocol` 加 `ClientMsg::Craft{recipe_id}`、`ws.rs` 收下、`game.rs` 查
   `crafting::recipe_by_id` 扣背包加道具。背包已持久化+已廣播,**不動快照 shape** → 最低風險。
2. **再 land 工具用途(1-D)**:`tools.rs` 已建,合成出的鎬子讓採礦更快。閉合
   「素材→合成→更快採」第一個完整正回饋圈,讓合成出來的東西**真的有用**。
3. **擴地(0-G-O2,乙太買地)**:摘 `economy.rs`(已在 main)。per-player fields 已是 Vec 廣播,
   能複用既有 shape 就低風險先上;需動 shape 才走 PR。

## 接下來 1-3 個切片(由上往下,每個附「驗收=玩家能做到什麼」)

1. **合成 MVP(1-C)— 最低風險、先 land**:木×3+石×2 → 鎬子,不動廣播 shape。
   **驗收 = 玩家在合成台把背包素材(採集+打怪掉落)合成出鎬子、囤積終於有去處**(經濟迴圈起步)。
2. **工具效用(1-D)— 閉環**:有鎬子採礦更快、沒有就慢。先只做鎬子×採礦一條。
   **驗收 = 合成出的鎬子真的讓採集變快**,第一個完整正回饋圈。
3. **擴地 MVP(0-G-O2)— 乙太的去處**:收成/打怪的乙太換一塊相鄰新地塊。薄:固定價、固定下一格、無市場。
   **驗收 = 玩家花乙太多拿一塊地、填滿後不再停滯**。
