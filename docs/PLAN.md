## 接下來的切片(薄切片快速連發,由上往下)

1. **薄背包:採集 → 進背包 → 看得見**(玩法鏈這條,backend 主力)
   驗收 = 玩家走近採集節點 → 物品進可開關「背包」面板 → 跨重連還在。
   只做這最小迴圈,**別**一次把合成/裝備/堆疊全做進來。
2. **改名薄切片**(高優先、並行):設定面板加改名輸入框 → 後端 `PATCH /api/profile`(過
   `sanitize_name`)→ 持久化 users.jsonl/DB → 即時反映 HUD/聊天/快照。
   驗收 = 玩家改名後自己與他人即時看到、重連仍是新名。
3. **物品有用途**:採到的東西能幹嘛(賣乙太 / 餵作物 / 當材料),讓背包不只是清單。
4. **戰鬥 MVP(自動打怪)**:世界有可打目標、角色自動攻擊附近敵人、掉落進背包。先最薄一條。
5. **(並行,別擋路)經濟 sink + 持久化**:乙太擴地、背包/戰鬥狀態存 Postgres——能先上就先上。

## 給各線的具體指示(分工清楚 = 少撞檔)

- **backend(主力)**:接 `gather`/`inventory` 進權威伺服器(Player 帶背包、`ws.rs` 加採集動作、
  快照帶背包 + 撒佈節點)。**動 live 廣播 shape → 走 PR、不自走 merge、前後端同版上線、別在
  共用工作樹留半成品**(比照 per-player 慘痛經驗)。持久化並行補。
- **frontend**:`web/game.js` 做**背包面板** + 採集回饋;另做**改名設定面板**(高優先)。
  新快照欄位一律 `|| []` 防呆。打磨暫緩,力氣全給新玩法 + 改名。
- **feature**:做玩法「內容」——可採集的東西怎麼擺、敵人種類/掉落,盡量寫新檔,別跟 backend
  搶同一段 ws 接線。改名後端(`PATCH /api/profile` + 持久化)也可由此線承接。
- **feedback**:挑「讓新玩法更好懂」的小提示。當前可直接做:**種田狀態顏色不好分辨**
  (`at=1780648665520`)——調高各生長階段色相對比即可。
