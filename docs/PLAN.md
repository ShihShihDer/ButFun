## 🎯 現在主攻:合成(1-C)接線——玩法鏈唯一還沒接通的那一環

**戰鬥/自動打怪(1-F)已完整上線**(2026-06-06 確認在 main):敵人廣播(`EnemyView`)、玩家靠近
`attack_nearest` 自動攻擊、敵人 `threat_at` 反擊、玩家 `Vitals` 扣血/被打趴/休息復原、打倒掉落進背包、
前端血條+HP HUD+受擊播報全有。**採集(`Gather`)也已接線**。玩法鏈現已連到:採集/打怪→背包→自動打怪有來有回。

**唯一斷點是「去處」**:三來源(農地/採集/打怪掉落)都在猛灌背包+乙太,卻沒地方花——
- **素材(木/石/廢料/掉落)沒去處** → 合成(1-C)沒接:`protocol` 無 `ClientMsg::Craft`、`ws.rs` 沒收、
  `game.rs` 沒呼叫 `crafting`。**但周邊全就緒**:`crafting.rs`/`tools.rs` 純邏輯在 main、背包已接 PG+已隨快照廣播、
  **前端合成台面板已建好且 `updateCraftPanel` 已依背包反灰**。缺的只剩那根 handler。
- **乙太沒去處** → 擴地(0-G-O2)沒接:`economy.rs` 在 main、前端買地面板已備(送 `buy_expansion` 意圖),
  但 `protocol`/`ws` 沒收。

GDD 白紙黑字「乙太有產出也要有去處」此刻是**最大缺口**。合成是玩法鏈斷點、且 additive 零契約變更(背包走既有快照)
→ **風險最低、最快見效**,本輪維護者第一優先。

### 給維護者的接線優先序(一次一條,別並行撞 main)
1. **先 land 合成(1-C)**:`protocol` 加 `ClientMsg::Craft{recipe_id}`、`ws.rs` 收下、`game.rs` 查
   `crafting::recipe_by_id` 扣背包加道具。背包已持久化+已廣播,**不動快照 shape** → 最低風險。
2. **再 land 工具用途(1-D)**:`tools.rs` 已建,合成出的鎬子讓採礦更快(`gather_speed_multiplier`)。
   閉合「素材→合成→更快採」第一個完整正回饋圈,讓合成出來的東西**真的有用**(否則鎬子是死道具)。
3. **擴地(0-G-O2,乙太買地)**:`economy.rs` 已建、前端面板已備。`protocol` 加 `BuyExpansion`、
   `ws`/`game` 收下扣乙太擴地。per-player fields 已是 Vec 廣播,能複用既有 shape 就低風險先上;需動 shape 才走 PR。

## 接下來 1-3 個切片(由上往下,每個附「驗收=玩家能做到什麼」)

1. **合成 MVP(1-C)— 最低風險、先 land**:木×3+石×2 → 鎬子,不動廣播 shape。
   **驗收 = 玩家在合成台把背包素材(採集+打怪掉落)合成出鎬子、囤積終於有去處**(經濟迴圈起步)。
2. **工具效用(1-D)— 閉環**:有鎬子採礦更快、沒有就慢。先只做鎬子×採礦一條。
   **驗收 = 合成出的鎬子真的讓採集變快**,第一個完整正回饋圈。
3. **擴地 MVP(0-G-O2)— 乙太的去處**:收成/打怪的乙太換一塊相鄰新地塊。薄:固定價、固定下一格、無市場。
   **驗收 = 玩家花乙太多拿一塊地、填滿後不再停滯**。
