## 🎯 現在主攻:戰鬥 / 自動打怪 接線(1-F)

採集→背包已上線(PR #18),但**背包目前是死路**——採到的東西沒地方用、世界裡也沒有會動的東西。
繼採集後**最大的可玩鉤子=讓世界活起來**:有敵人在晃、走近自動打、掉落直接進背包。
零件已齊(`auto/feature` 的 `combat.rs`/`enemy_field.rs` 純邏輯早建好沒接線),**照採集→背包同法接**,快。
> 為什麼這個最提升可玩性:它一次給「新活動 + 背包有產出 + 世界有生氣」三件事,且接線而非從零寫。

## 接下來 1-3 個切片(由上往下)

1. **戰鬥 MVP / 自動打怪**:把 `combat.rs`/`enemy_field.rs` 接進 `ws.rs`/`game.rs` + 快照帶 `enemies`,
   前端畫敵人 + 血條 + 受擊/擊殺飄字。**驗收 = 世界有敵人會動、走近自動攻擊、擊殺掉落進背包。**
   薄:純記憶體、固定幾隻、無重生平衡,先讓「能打」上線,平衡/重生隨後補。
2. **物品有用途(合成/賣乙太)**:接 `crafting.rs`(1-C 配方已建),木/石 → 合成物或換乙太。
   **驗收 = 玩家能把背包素材變成有用的東西**,背包不再只是清單、乙太有了去處(經濟迴圈閉環)。
3. **改名(並行、最小、玩家多次點名)**:`PATCH /api/profile` + 設定面板 + live 廣播。
   **驗收 = 改名後自他即時看到、重連仍新名。** 不在玩法鏈上,但薄又被點名,空檔就接。

## 給各線的具體指示

- **feature**:**別再堆「不接線」的新純邏輯**(戰鬥/合成/工具/載具/改名都建好沒接,已遠超能接的速度)。
  改做能**獨立 land** 的小事:敵人**資料**(種類/血量/掉落表/出沒點)、合成**配方資料**。接線交維護者。
- **frontend**:做能獨立 land 的小事——**敵人 sprite / 血條 / 受擊飄字的畫法**(本輪主攻會用到,先備好)、
  背包面板打磨。別再堆農地微打磨。分支卡衝突就從 main 重開乾淨枝。
- **backend**:小而獨立——改名 `PATCH /api/profile` 收尾、持久化補平(背包/節點存 Postgres)。
  大接線(戰鬥/合成進 ws.rs)由**維護者一次一條**做,別搶。
- **feedback**:挑「一眼看懂新玩法」的小提示——敵人「走近會自動打」、擊殺掉落去哪、背包素材能幹嘛。
