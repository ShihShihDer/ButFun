## 🎯 現在主攻:戰鬥 / 自動打怪 接線(1-F)— 仍未動,這是最高優先

採集→背包已上線(PR #18),改名也已完整 merge(PR #19)。但**背包仍是死路**——採到的東西
沒地方用、世界裡也沒有會動的東西。繼採集後**最大的可玩鉤子=讓世界活起來**:有敵人在晃、
走近自動打、掉落直接進背包。零件 **100% 齊全且去風險**:`auto/feature` 的 `combat.rs`
(敵人/扣血/掉落/重生)、`enemy_field.rs`(世界佈置+走近自動鎖定)、`vitals.rs`(玩家生命/
被打趴休息復原)、威脅聚合,**還附跨模組組合測試鎖住接縫**。**照採集→背包同法接即可,純接線、
不必造任何零件。**
> ⚠️ 注意:近 5 個 commit 全是「採集」的報讀器/提示微打磨,**沒人接戰鬥這條線**。

> 為什麼最提升可玩性:一次給「新活動 + 背包有產出 + 世界有生氣」三件事,零件齊+測試在,接線最快。

## 給各線的具體指示

- **feature**:**停堆純邏輯**。戰鬥/合成/工具/載具全建好沒接,再加只是囤貨。**只做能獨立 land
  的小事**:敵人**資料**(種類/血量/掉落表/出沒點)、合成**配方資料表**、工具效用**數值表**、組合
  測試補強。新純邏輯系統一律別開,接線交維護者。
- **frontend**:做能獨立 land 的小事——**敵人 sprite / 血條 / 受擊飄字的畫法**(本輪主攻會用到,
  先備好)、合成台/背包面板雛形。別再堆採集/農地微打磨。分支卡衝突就從 main 重開乾淨枝。
- **backend**:小而獨立——背包/採集節點持久化補平(存 Postgres,沿 0-E)。**大接線(戰鬥/合成
  進 ws.rs)由維護者一次一條做,別搶。** 改名已完整上線,不必再碰。
- **feedback**:**採集的報讀器/提示已夠多,先收手**。挑「一眼看懂新玩法」的小提示——等戰鬥上線後
  補「敵人走近會自動打」「擊殺掉落去哪」;合成上線後補「素材能合成什麼」。

- **繼續微打磨採集/農地的提示與報讀器**——已足夠,再磨是範圍蔓延,把空間讓給戰鬥/合成。
- 改名——已完整 merge(PR #19),完成,別再動。
- 救舊衝突 PR——別投時間 rebase,從 main 重開乾淨小枝。
- 載具/飛船/多星球/競速——真正遠期(工具 1-D 之後再談載具 1-E)。
