### 🔑 關鍵新洞察:戰鬥卡死的真正瓶頸是「動 live 廣播 shape」——而合成不用動它
戰鬥要在快照塞新的 `enemies` 陣列 = 架構級廣播契約變更 = 正是本 repo 一直 land 不了的那種。
**但合成(1-C)不需要動廣播 shape**:背包已隨快照廣播(PR#18),合成只是「client 送 `Craft`
→ 伺服器查配方扣素材加工具 → 背包變動走**既有**快照」,前端只多一個面板(純前端、零契約變更)。
**所以合成是比戰鬥更好 land 的薄切片——先 land 它,別讓玩法鏈整輪原地踏步。**

### 給維護者的優先序(本輪重點調整)
1. **戰鬥仍是想要的方向**(總監點名):這輪若有空做架構級接線,照上輪解法——從 main 開乾淨枝、
   摘 `combat.rs`/`enemy_field.rs`/`vitals.rs` 三個新檔(additive、不碰 PR#17 對既有檔的過時編輯)、
   加 `enemies` 快照欄位(前端 `|| []`)、開 PR 給審。
2. **但別讓整條鏈再卡 5 輪**:若戰鬥這輪又沒空接,**改先 land 合成(1-C)**——它不動廣播 shape、
   風險低、同樣推進玩法鏈(素材終於有去處、經濟迴圈起步)。寧可合成先上,也別讓「可玩」整輪踏步。

## 接下來 1-3 個切片(由上往下)

1. **合成 MVP(1-C)— 最低風險、先 land 它**:接 `crafting.rs`(配方已建,摘檔同採集法),
   木×3+石×2 → 鎬子。**不動廣播 shape**(背包已在快照)。**驗收 = 玩家能把背包素材合成出鎬子、
   素材有去處**(經濟迴圈起步)。
2. **戰鬥 / 自動打怪(1-F)**:世界有敵人會動、走近自動打、擊殺掉落進背包、被打趴會休息復原。
   需動廣播 shape(加 `enemies`)→ 開 PR。**驗收 = 世界終於「活」起來**,直接解玩家「太靜/沒事做」。
   薄:純記憶體、固定幾隻、無重生平衡,先讓「能打」上線。
3. **工具效用(1-D)— 閉環**:身上有鎬子採礦更快、沒有就慢(`tools.rs` 已建)。
   **驗收 = 合成出的鎬子真的讓採集變快**,形成第一個完整正回饋圈。薄:先只做鎬子×採礦一條。
