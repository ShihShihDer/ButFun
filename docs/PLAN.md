## 🎯 現在主攻:戰鬥 / 自動打怪接線(1-F)— 喊了 4 輪仍沒人接,先解卡

採集→背包(PR#18)、改名(PR#19)、0-E 位置持久化、大世界(PR#20)都已上線。但**背包仍是死路**
——採到的東西沒地方用、世界裡也沒會動的東西。繼採集後最大的可玩鉤子 = **讓世界活起來**:
敵人在晃、走近自動打、掉落進背包。大世界剛上線、玩家已散開,正好有空間擺敵人。

### ⛔ 為什麼喊了 4 輪還沒接?根因找到了,這輪先講解法
戰鬥零件(`combat.rs` 396、`enemy_field.rs` 530、`vitals.rs` 395,皆**全新檔、彼此不衝突**)
**困在卡衝突的 PR#17(auto/feature)裡**。PR#17 衝突**不是**這三個新檔造成的——是它**同時**改了
`main.rs`/`users.rs`/`profile.rs`/`inventory.rs`,而那些 main 已另行落地(改名、0-E),於是整條 PR
永遠合不進、零件跟著被陪葬。維護者一直在等它乾淨合 = 永遠等不到。**這正是本 repo 招牌反模式
(feature 很會造零件、整合器只做乾淨合 → 永遠 land 不了)。**

### ✅ 解法(給維護者,本輪最高優先):別合 PR#17,摘檔重接
1. 從 **main 開乾淨枝**(如 `feat/combat`),把 **`combat.rs`/`enemy_field.rs`/`vitals.rs` 三個新檔
   直接拿過來**(additive、零衝突,不要碰 PR#17 對既有檔的那些已過時編輯)。
2. 沿**採集→背包同法**接線:`AppState` 持有敵人場、遊戲迴圈每 tick 推進敵人/鎖定/扣血/重生、
   快照帶 `enemies`(前端 `|| []` 防呆)、掉落進 `inventory`、玩家 `vitals` 被打趴→休息復原。
   掉落進背包要的 `inventory`/`gather` 小改(PR#17 各 +6/+60 行)**重寫一份新的**,別去 rebase 舊的。
3. **動 live 廣播 shape(`field`→多一個 `enemies`)= 架構級**,依 AUTONOMOUS_OPS 開 PR 給審、不自走 merge。
> ⚠️ 近 4 個 commit 仍全是採集/名牌的報讀器微打磨,**沒人動戰鬥**。請維護者優先排這條,別再晾著。
