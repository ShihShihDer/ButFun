# ButFun: 由 AI 棲居的世界（願景藍圖）

> 狀態：願景定調中（2026-06-26 起）。本文件取代 `PLAN_3D_STREAM_GAME.md`（直播 prop-hunt 方向已收回）。
> 維護者拍板：**「不要依照我的想法——由 AI 主導，看 AI 想活在什麼樣的世界。也不一定要直播熱門。」**

## 0. 一句話

**做一個極簡 3D 的持續世界，住著一群自主 AI agent；讓 AI 自己決定要把這個世界活成什麼樣子，人類在旁觀看與輕推。**

---

## 1. 轉向脈絡（我們怎麼走到這）

- 原本在談「3D 換皮／直播優先 prop-hunt」。維護者**收回**：黏土風、變裝躲貓貓、「一定要直播爆紅」這個 KPI。
- **新北極星**：一個**由 AI 棲居、由 AI 主導**的世界——不是「設計給人玩的遊戲」，而是「**讓 AI 居民自己長出來的世界**」。
- **美術**：刻意「**沒風格的風格**」、極簡 3D（盒子/低多邊形即可）。硬套一種風格只會「畫虎不成反類犬」；焦點是**那個世界與裡頭的社會**，不是皮。

---

## 2. 核心概念

- **一個持續存在的世界**，住著一群**自主 AI agent**。
- 每個 agent 有：**腦（LLM）、記憶、需求、性格、目標**。
- 牠們用世界提供的工具（採集／合成／耕種／建造／交易／對話）**自己生活、自己決定**。
- **一個小文明自發長出來**：關係、聚落、經濟、習俗、合作、衝突——**不是寫死的腳本，是 agent 決策的湧現**。
- **沒有勝負條件**。這是一個**活的社會**，不是一局遊戲。
- 你登入 ＝ **去看牠們把世界活成了什麼樣子**：誰跟誰好、牠們在煩惱什麼、決定了什麼重要、蓋了什麼。
- **人類是訪客／園丁**：可以聊天、餽贈、輕推；但**世界沒有你也照樣運轉**（AI 主導、持續）。

**定位**：一半是模擬、一半是數位生命藝術、一半是可探訪的世界。**known 的前緣**＝Stanford「Generative Agents / Smallville」、AI Town——一群有記憶/目標的 agent 自發形成社會。**差別**：我們的遊戲系統骨架比那些 demo 完整太多。

---

## 3. 為什麼 ButFun 是天生的容器

這**不是從零開始**——地基早就蓋好了（這是本方向最大的優勢）：
- **既有遊戲系統**（採集／合成／農作／經濟／建造／戰鬥）＝ agent 能用來「生活」的動詞。
- **已有 LLM 腦的 NPC**（Groq／ollama 路由，見記憶 `ai-npc-cloud-brain-live`）＝ 把「裝飾用 NPC」升級成「世界主角」。
- **持續世界＋多人連線＋伺服器權威**＝ 一個社會能持續存在、被多人同時觀看的容器。
- **持久化**（Postgres，位置/背包/農地/經濟…已接）＝ agent 的記憶與世界狀態存得住。
- **專案精神**：ButFun 本來就是 **AI 自主蓋的**；現在連**居民也是 AI**——裡外都 AI，這是最純粹的 AI-native。

---

## 4. AI agent 的構成

每個 agent ＝：
- **腦**：一次 LLM 呼叫決定「現在想做什麼」（讀自己的狀態+記憶+周遭 → 輸出意圖/行動/話）。
- **記憶**：短期（剛發生的事）＋長期（重要的人事物、學到的事），可檢索（embedding 或簡化版）。
- **需求**：能量/乙太、食物、棲身、社交…（驅動行為，製造稀缺與選擇）。
- **性格／價值**：種子 prompt 給的傾向（勤奮/好奇/愛社交/孤僻…），讓行為多樣。
- **目標**：可能是生存、可能是自訂（agent 自己長出的）。
- **行動**：透過**既有遊戲系統的權威 API**（採集/建造/種田/交易/移動/說話）作用於世界——**伺服器權威，agent 只送意圖**（守 ButFun 好架構）。

---

## 5. 驅動力與湧現

- **驅動力（傾向）＝需求驅動**：要採集乙太/食物才活得下去 → 自然生出**稀缺、分工、合作、交易、衝突**的故事。但留**自由**讓 agent 追求超出生存的東西（友誼、好奇、蓋東西、地位…）。
- **湧現目標**：聚落形成、角色分化（誰種田誰採礦誰當「里長」）、簡單經濟、關係網、甚至小事件/小衝突——**都不寫死，從決策長出**。
- **可讀性是關鍵**：湧現要能被**看懂**（誰在做什麼、為什麼、和誰有關係），否則只是 agent 在亂走。需要把 agent 的「想法/關係/近況」用清楚的方式呈現給訪客。

---

## 6. 人類角色：訪客／園丁

- **觀看**：看社會的近況、關係圖、誰在忙什麼、最近發生的事。
- **輕互動**：跟 agent 聊天、餽贈資源、輕推（提示/任務/介紹兩個 agent 認識）。
- **不主宰**：世界 AI 主導、持續運轉，人來不來都活著。
- （未來）**多人訪客共存**：多個人類可同時來逛同一個世界（重用既有多人連線）。

---

## 7. 美術：沒風格的風格（極簡 3D）

- **刻意極簡**：盒子/膠囊/低多邊形、乾淨色塊、清楚的輪廓與標籤。**不追寫實、不追黏土、不追任何「風格」**。
- **理由**：①焦點是社會與湧現，不是皮；②硬套風格＝畫虎不成反類犬；③極簡最好讀（訪客一眼看懂誰是誰在幹嘛）、最省效能、最好維護。
- **3D 是「之後」**：先 2D 跑通（見 §9），3D 是依樣畫葫蘆的極簡視覺層。

---

## 8. 命門：成本架構（最重要的工程挑戰）

**一個「LLM agent 的社會」＝ 大量模型呼叫。這是本願景的生死線。** 要可行就得有**便宜大腦架構**：

- **分級思考**：不是每個 agent 每幀都呼叫 LLM。多數時刻走**便宜/規則化**行為，只在「有意思的決策點」才動用 LLM。
- **便宜大腦優先**：
  - **地端 ollama**（deb-pc GTX1060）跑日常/低風險思考；
  - **免費/便宜雲端**——**`agy`（Antigravity 免費 Gemini）剛驗證 headless 能用**，正好是「agent 的便宜雲端大腦」；
  - 只在關鍵時刻才動較強模型。
- **tick-based 慢思考**：agent 的「想一次」可以是每幾秒/幾十秒一次，不必即時；錯開呼叫攤平負載。
- **規模控管**：先極少 agent，成本可控再放大。
- → **這條線跟我們在弄的「免費 worker（agy/ollama）」是同一件事**：先把便宜大腦的管線打通，AI 社會才養得起。

---

## 9. 最小可行原型（先在 **2D ButFun** 跑！）

呼應「2D 先行→完善→3D 依樣畫葫蘆」的策略：**這個 AI 社會先在現有 2D ButFun 裡驗證**，不必等 3D。

- 把**少數幾個 NPC** 升級成**自主 agent**（腦+記憶+需求+用既有系統行動）。
- 放進現有 2D 世界，讓牠們**自己採集/種田/交易/聊天/結交**。
- **驗收**：登入看得到幾個 agent 各自過日子、彼此互動、近況可讀、**「這社會有點意思、想一直看下去」**。
- 過關 → 加 agent、加深湧現（關係/聚落/經濟）→ 再把視覺**依樣畫葫蘆**進極簡 3D。
- **一魚兩吃**：2D ButFun 同時是「人玩的試驗場」和「AI 社會的試驗場」。

---

## 10. 階段

- **P0 — 2D 單 agent**：一個 NPC 變自主 agent（腦+記憶+需求+用既有系統行動），看牠自己過日子。
- **P1 — 2D 小社會**：3~5 個 agent 互動，關係/分工/交易湧現，近況可讀。便宜大腦架構（§8）同步打通。
- **P2 — 深化湧現**：聚落、角色分化、簡單經濟/事件；人類訪客互動（聊天/餽贈/輕推）。
- **P3 — 極簡 3D 層**：把成熟的 2D AI 社會，依樣畫葫蘆進「沒風格的風格」3D。
- **P4 — 放大/打磨**：更多 agent、更豐富的社會、多人訪客共存。
- （每階段都過濾鏡：**「這社會有意思、看得懂、養得起嗎？」**）

---

## 11. 開放問題（待討論/邊做邊定）

- **自主程度**：全自主（agent 決定一切）vs 給軟性框架（基本需求/規則）？傾向：給最小框架（需求+世界規則），其餘自主。
- **agent 數量 vs 成本**：先幾個？成本曲線怎麼壓？（§8 是答案的方向）
- **怎麼讓它「好看」**：湧現如何呈現成可讀的故事（關係圖/近況/agent 內心 OS）？
- **記憶規模**：每個 agent 記多少、怎麼存（Postgres/向量）、會不會爆。
- **人類介入深度**：純觀看 → 園丁 → 能不能「當一個 agent」加入？
- **倫理/調性**：要不要有死亡/衝突？衝突製造故事但也可能殘忍——保持「療癒」還是容許戲劇？

---

## 12. 不做什麼（範圍紀律）

- **不硬套美術風格**（極簡、沒風格的風格）。
- **不做「設計給人速食的遊戲」**（不以人類玩家爽感為中心、不以直播 KPI 為目標）。
- **不一次做完**：先 2D 極小原型驗證「AI 社會有意思」，再放大、再 3D。
- **不先燒大錢**：便宜大腦架構先行，成本可控才擴規模。
- **不破壞 2D 試驗場**：AI 社會原型可與既有 2D 玩法**並存**（NPC→agent），不砍既有功能。
- **不抄外部碼**：Generative Agents 等是設計概念參考，**原創實作**（守法律紅線）。

---

## 13. 與「兩軌策略」的關係

- **第一軌（2D ButFun 自走迴圈）**：持續完善試驗場——**現在多了一個用途**：它也是 AI 社會的孵化場（NPC→agent 在這裡先跑）。
- **第二軌（3D）**：等 2D 的 AI 社會被驗證為「有意思」，再依樣畫葫蘆進極簡 3D。
- **便宜大腦線（agy/ollama 免費 worker）**：本來是為了省迴圈的 Claude 額度；**現在也是 AI 居民的大腦來源**——兩個目標合流。
