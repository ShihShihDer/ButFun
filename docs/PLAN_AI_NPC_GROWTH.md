# AI NPC 增量建造計畫（給自走迴圈一塊一塊長）

> 北極星與設計原則見 [VISION_AI_EMERGENT_ECOSYSTEM.md](VISION_AI_EMERGENT_ECOSYSTEM.md)。
> 這份是**建造順序**：每個增量小、安全、可獨立上線。**一次一塊，照順序由上往下。**

## 現況（已上線 / 審查中）

- ✅ **會講話的商人**（PR #151）：走近商人打字 → 非同步呼叫地端 ollama（不卡 15Hz 迴圈）
  → 人設「薇拉」+ 世界觀回話。`src/npc_chat.rs`、協定 `TalkToNpc`/`NpcReply`。
- ✅ **個人記憶**：每位玩家對 NPC 一句印象，對話後濃縮更新（`NpcRel`，記憶體 v1）。
- ✅ **第一隻手（自主送禮，無寫死門檻）**：往來統計只當「資料」餵給 NPC，**送不送他自己判斷**
  （回話夾暗號 `[GIFT]`）；引擎只管上限。
- ✅ **約束=真實稀缺**（PR #153 已 merged）：送禮從 NPC「有限的餘裕」`npc_gift_stock` 扣，
  送完就沒；餘裕餵進 prompt 讓他餘裕少時自然更挑。
- ✅ **六大 NPC 全活起來**（PR #154 已 merged）：工匠老胡、獵手蘭卡、探勘員芙利亞、採購代理人吉爾、評審老農加入 AI 對話。
- ✅ **LLM 限流佇列**（ROADMAP 59，本輪）：全域 Semaphore(5) + 每人每 NPC 冷卻 8 秒，多人同時聊 NPC 不再打爆 CPU。

## 不可破的安全護欄（每個增量都要守，違反就退回）

1. **腦子自由、手有界**：LLM 只生成「對話文字 / 受限決策」，**碰不到遊戲狀態**；所有實際
   效果由引擎執行＋驗證，只能在白名單動作內。
2. **約束=世界規則本身**：NPC 能給的，只能是他**實際擁有**的（金庫/庫存/餘裕），**不能無中生有**。
3. **沒有寫死門檻**：別做「滿 N 次→自動 X」。統計只當資料餵 NPC 判斷，效果由他決定、引擎管上限。
4. **玩家資料絕不被毀**；療癒世界**不搞永久死亡**（回歸乙太）。
5. **非同步**：LLM 呼叫一律 `tokio::spawn`，**永不阻塞 15Hz 權威迴圈**。
6. **降級安全 + env gate**：`BUTFUN_NPC_LLM=1` 才啟用；連不到 ollama 回罐頭。**CI 沒有 ollama，
   純邏輯要可測、LLM 呼叫不進測試**。prod 預設關＝零風險。
7. **記憶防污染**：公共記憶=引擎事實（玩家寫不進）；個人記憶隔離（A 講 B 壞話不影響 B）。

## 建造順序（由上往下，一次一塊）

1. ✅ **LLM 呼叫限流 / 佇列**（ROADMAP 59 已完成）：全域 Semaphore(5) + 每人每 NPC 冷卻 8 秒。
2. ✅ **記憶 + 餘裕持久化**（ROADMAP 60 已完成）：`NpcRel` 與 `npc_gift_stock` 升到 Postgres 持久化。
   migration 0016 向後相容；`src/npc_memory_store.rs` 仿 land_plot_store 模式；啟動時載回、對話後 fire-and-forget upsert。
3. ✅ **關係綁真實交易**（ROADMAP 61 已完成）：`NpcRel` 新增 `sell_count`/`buy_count` 引擎事實統計；
   `ShopSell`/`ShopBuy` 成交後累積，持久化 Postgres（migration 0017）；`system_prompt` 餵真實交易統計給 NPC。
4. **餘裕回補=補貨**：`npc_gift_stock` 隨遊戲時間慢慢回補；進階：餘裕來自商人**實際賺到的**
   乙太/庫存（給出去的是他賺來的），讓慷慨有真實來源與代價。
5. **更多手（每隻都綁真實資源）**：商人自己決定要不要給熟客折扣（他少賺）、要不要進某種貨。
   每個動作走白名單 + 引擎驗證 + 由 NPC 判斷時機。
6. **第二個 NPC：里長/小領主**，有自己的小金庫，能資助村裡的小事（修補、辦活動）——金庫見底就做不了。
7. **分層模型路由**（見願景文件）：地端小模型常駐、難題自我求救升級；之後加不同模型給不同 NPC。
8. **公共記憶=引擎世界事件**：裂縫開啟、某人擊敗精英…注入共享「世界近況」，NPC 對話會提到（玩家寫不進）。
9. **生命週期（回歸乙太）**：老化→平靜回歸乙太→收徒繼承（接手店/職位 + 口述歷史式記憶繼承）。
10. **湧現派系**：多個有需求的 agentic NPC，聯盟/競爭/背叛自己長出來。
11. **（可能，非必須）律法**：湧現治理——派系利益衝突後規則自然冒出。玩家說「可能」，到那步再看。

## 給迴圈的交接提醒

- 每塊都要：`cargo build`/`cargo test` 全綠、附純邏輯測試、開 PR 走審查閘（**別自合**）。
- 動到玩家資料／架構級的（如持久化 migration）只開 PR、不自 merge。
- 每個 PR 都要能讓玩家「看得出 NPC 更活了」一點，且守上面七條護欄。
- 真正讓 NPC 用 AI 思考需在主機開 ollama + `BUTFUN_NPC_LLM=1`——那是維護窗/人的動作，不是迴圈。
