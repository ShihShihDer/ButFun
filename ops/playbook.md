# Playbook — 接案標準作業流程

> 一人公司的 SOP。看到這份檔案就知道每一步該做什麼。

---

## 1. 案件評估流程 (Lead Evaluation)

當你貼一個案件到 `inbox/` 後,我會用以下框架產出 `leads/案件代號.md`:

### 1.1 紅旗檢查 (Deal-breakers — 中任何一項就拒絕)

- [ ] 客戶要求**先做後付**且預算 > 1 天工
- [ ] 客戶**不肯說公司/個人身分**
- [ ] 案件涉及**詐騙、博弈代操、學術代寫、爬取版權內容**
- [ ] 客戶要求**繞過平台私下交易** (新客戶禁止,違反 TOS 易被封號)
- [ ] 客戶 review 數 = 0 且不肯先付小訂金
- [ ] 規格極度模糊但要求「fixed price 含修改不限」
- [ ] 客戶 timezone / 語言對不上,但要求 24h 即時回應

### 1.2 黃旗 (要加 buffer 或加價)

- 「應該很簡單」「就只是個小功能」← 通常不簡單
- 已經換過至少 1 個前任工程師
- 截止日 < 估時 × 1.5
- 客戶不熟技術但堅持指定技術選型

### 1.3 評分 (1-5)

| 維度 | 權重 |
|---|---|
| 技術可行性 (Claude Code 能不能做) | × 2 |
| 預算合理性 (估時 vs 報酬) | × 3 |
| 客戶品質 (溝通、付款記錄、態度) | × 2 |
| 戰略價值 (能不能變 portfolio / 長期客戶) | × 1 |

**總分 ≥ 24 → 強烈建議投; 18-23 → 投; < 18 → 跳過**

---

## 2. 估時公式

```
總工時 = (Claude Code 開發時數 × 1.0) + (你的審閱/溝通時數 × 1.0) + (整合/部署/QA × 0.5)
報價工時 = 總工時 × 1.4   ← 40% buffer 給未預期狀況
```

### 常見任務參考估時 (用 Claude Code)
| 任務 | 工時 |
|---|---|
| Claude API 整合到現有 web app | 4-8 h |
| 全新 CRUD web app (5-10 個 entity) | 16-32 h |
| AI agent + 工具整合 (3-5 個 tool) | 12-24 h |
| 既有 Spring Boot 系統加新功能 | 4-12 h |
| Jenkins / CI 設定 | 2-6 h |
| K8s deployment 從零 | 8-16 h |
| iOS App MVP (3-5 screen) | 24-48 h |
| 爬蟲 + 排程 + 通知 | 6-12 h |
| Discord / Slack / LINE bot | 4-10 h |

## 3. 報價策略

### 3.1 三段報價法 (Anchor + Target + Floor)

每次給客戶 3 個選項:

- **Premium (anchor, 1.5x)**: 加上「保固 1 個月、優先支援、文件齊備」
- **Standard (target, 1.0x)**: 你真正想接的價
- **Lite (floor, 0.7x)**: 砍掉文件/QA/保固,只交付能跑的版本

→ 客戶通常選 Standard,你拿到目標價而非被殺到 Floor。

### 3.2 不要做的事
- ❌ 第一次溝通就報價 (先問清需求)
- ❌ 用「時薪」報固定案 (報固定價,內部用時薪算)
- ❌ 同意「先看到才付」(永遠要訂金或 milestone)

---

## 4. 接案後啟動 SOP

當客戶確定要合作:

1. **建立專案目錄**: `projects/案件代號/`
2. **開新 git 分支**: `client/案件代號`
3. **產出 brief.md**: 把所有客戶需求結構化
4. **訂 milestone + 付款條件**(寫進 `decisions.md`)
5. **第一次交付前要求訂金** (建議 30-50%)
6. **建立 daily log** 在 `timeline.md`

---

## 5. 溝通模板規則

- **回應時間承諾**: 工作日 < 8h, 週末可不回
- **狀態更新頻率**: 每 2-3 天主動更新進度,即使沒大進展
- **壞消息**: 24h 內告知,附**問題 + 影響 + 3 個選項**,不要只丟問題
- **scope creep**: 任何「順便加一下」都用「這算 change request,評估後加報價」回應,不要默默吞

---

## 6. 收款規則

| 案件規模 | 付款方式 |
|---|---|
| < USD 200 / NT$ 6,000 | 100% 預付 |
| USD 200-1,000 | 50% 訂金 + 50% 交付 |
| USD 1,000-5,000 | 30% 訂金 + 30% 中段 + 40% 交付 |
| > USD 5,000 | 拆 4-5 個 milestone,每個獨立驗收 |

**平台案**: 走平台 escrow,不走外部
**外部案**: Wise / Payoneer / 銀行 SWIFT,**金額 > USD 500 一定要簽簡易合約**
