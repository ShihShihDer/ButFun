# Proposal 模板 — 中文

> 每份提案都要**針對個案客製化**。罐頭文會被忽略。
> 目標長度: **300-500 字**。中文案主比英文案主更吃「親切感 + 專業感」並重。

---

## 結構 (5 段)

### 第 1 段 — 證明你看過需求 (1-2 句)
> 引用客戶 brief 裡的**具體細節**,證明你不是亂貼。

### 第 2 段 — 你的背景 (2 句,不要列履歷)
> 一個跟這個案子相關的過去經驗 + 數字。

### 第 3 段 — 怎麼做 (條列 3-4 點)
> 證明你有思考過,不是接案就動手。

### 第 4 段 — 為什麼選你 (AI 加速)
> 直接說「我有 Claude 當工程助理,交付速度是傳統工程師的 3-5 倍」。台灣客戶現在對 AI 接受度高。

### 第 5 段 — 引導下一步
> 問一個具體問題,逼客戶回覆。

---

## 完整範例

```
您好 [客戶稱呼],

看到您提到目前的 LINE Bot 推播會在流量大時掉訊息 — 這通常不是 LINE API 的問題,而是 webhook 的 backpressure 沒處理好。可以修。

我的背景: 之前在博弈科技做高併發開獎系統 (尖峰 10K+ req/s),目前在 SI 公司帶團隊做 iOS / IoT / 5G,主導 CI/CD 平台建置 (Jenkins + K8s + ELK)。

針對您的需求,我的做法是:
1. 先用一週的 log 確認掉訊息的 pattern (是 timeout、429、還是訊息順序錯)
2. 在 webhook 後面加 durable queue (Redis Streams 或 SQS) + idempotency key
3. 加上監控告警 (Sentry / 自建 dashboard),讓您比客戶更早知道問題
4. 交付物: PR + 部署文件 + 2 週的 bug fix 保固

差異化: 我用 Claude 當工程協作工具,所以同樣品質下交付速度是一般 SOHO 的 3-5 倍。但架構決策、資安判斷都是我親自把關,不是 AI 亂寫一通。

報價前想確認一下: 您的 LINE Bot 是 standalone 服務,還是嵌在現有的大系統裡? 這會影響部署方式跟報價。

期待您的回覆。

— [你的名字]
[聯絡方式]
```

## 中文案主常見地雷與應對

| 客戶說 | 真實意思 | 你的回應 |
|---|---|---|
| 「先做 demo 看看」 | 想白嫖 | 「demo 是另外的付費案,可以先簽 1 萬的 PoC」 |
| 「應該很簡單吧?」 | 預算很低 | 「簡單的話 1 天 NT$XX,複雜的話另算」|
| 「我們長期合作的話可以便宜嗎?」 | 想壓首單 | 「累積 3 案後享 9 折」 |
| 「我朋友報價只要一半」 | 砍價 | 「那您可以找他,我這邊是這個價,有需要再聯絡」 |
| 「不用簽合約啦,大家朋友」 | 想跑單 | 「制度問題,所有客戶都簽,內容很簡單」 |

## 不要做的事

- ❌「您好,我對您的案子很有興趣...」(罐頭開頭)
- ❌ 一次列出所有會的技術
- ❌ 第一次溝通就承諾完成時間
- ❌ 用「saoasi 這只是小 case 啦」這種輕浮口吻
- ❌ 報價低於台灣基本工資 (對自己跟同業都不負責)
